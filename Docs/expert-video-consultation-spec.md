# 专家视频会诊演示功能 Spec

> 版本：V1.0
> 日期：2026-07-14
> 范围：工程师端检修向导中的专家会诊演示；纯前端，不接入真实音视频。

## 1. 背景与目标

当前检修向导底部的“专家电话会诊”仅为预留按钮，无法支撑汇报时的完整演示。V1 将其改为可点击的“专家视频会诊”，以工业调度终端的视觉语言呈现专家选择、呼叫接入、会诊和结束四个环节。

本版目标：

- 点击按钮后可完成一条连续、可信、可重复演示的会诊流程。
- 自动携带当前设备、告警、场站和检修步骤，让会诊与主业务有明确关联。
- 视频画面统一使用黑屏占位，不请求摄像头或麦克风权限。
- 关闭或结束会诊后返回原检修向导，已勾选项和当前步骤不丢失。

## 2. 非目标

- 不接入 WebRTC、腾讯会议 SDK 或真实视频流。
- 不创建真实会议室，不请求浏览器媒体权限。
- 不保存聊天记录、录音、录像或真实通话数据。
- 不实现多端同步、呼叫推送和专家端接听。

## 3. 入口与命名

- 入口位置：检修向导底部操作区，保留在“完成并继续”之后。
- 原名称：`专家电话会诊 · 预留`。
- 新名称：`专家视频会诊`，移除“预留”标签。
- 图标：视频摄像机图标。
- 打开方式：全屏遮罩式会诊工作台；底层检修页面保持挂载。

## 4. 页面状态机

| 状态 | 用户看到的内容 | 可执行操作 | 退出结果 |
|---|---|---|---|
| `selecting` 选择专家 | 当前事件摘要、三位专家、推荐标识 | 选择专家、发起会诊、关闭 | 关闭后回到检修向导 |
| `connecting` 正在接入 | 黑屏视频区、接入进度、阶段文案 | 取消呼叫 | 取消后返回专家选择 |
| `connected` 会诊中 | 黑屏主画面、现场黑屏小窗、上下状态栏、右侧上下文、底部控制栏 | 静音、关闭摄像头、切换资料栏、结束会诊 | 点击结束进入二次确认 |
| `ending` 结束确认 | 结束会诊确认浮层 | 继续会诊、确认结束 | 确认后进入完成页 |
| `complete` 已结束 | 会诊时长、专家、上下文同步结果 | 返回检修向导 | 回到原步骤 |

## 5. 详细交互流程

### 5.1 选择专家

默认选中推荐专家“王海峰”，并展示：

- 姓名：王海峰
- 职称：高级工控设备检修工程师
- 机构：国家管网集团山东德州分输站 · 设备技术组
- 专长：工控机、站控系统、散热系统、工业通信
- 状态：在线 · 预计 10 秒接入

另提供两位演示专家，便于表现系统具备专家调度能力。专家卡片使用单选逻辑，未选择专家时不得发起会诊。

页面右侧展示自动携带的会诊上下文：

- 场站：山东德州分输站
- 设备：研华 ACP-4000 / IPC-610 工控机
- 关联告警：TEMP/FAN 告警，风扇转速偏低
- 当前步骤：取自检修向导当前步骤
- 进度：第 N / M 步

### 5.2 正在接入

点击“发起视频会诊”后进入约 2.4 秒的自动接入过程：

1. 创建临时会诊室；
2. 同步设备与告警上下文；
3. 呼叫所选专家；
4. 显示“专家已接入”并进入会诊主界面。

视频区始终保持黑屏。连接过程只使用边缘扫描线、状态点和文字变化，不伪造真实人物画面。

### 5.3 会诊中

主界面参考会议软件的信息结构，但不复制品牌资产：

- 顶栏：会诊标题、临时会议号、加密状态、网络状态、计时器。
- 主画面：纯黑专家视频区；左下显示专家身份牌；中央弱提示“专家视频画面 · 演示占位”。
- 现场小窗：右下角纯黑画面，标注“山东德州分输站现场终端”。
- 右侧资料栏：设备、告警、当前检修步骤、已同步资料状态。
- 底部工具栏：麦克风、摄像头、现场画面、会诊资料、结束会诊。

麦克风和摄像头按钮只改变界面状态，不调用浏览器媒体接口。默认麦克风开启、摄像头关闭，以符合本版黑屏演示策略。

### 5.4 结束与返回

点击“结束会诊”先显示二次确认，避免演示误触。确认后展示完成页：

- 会诊已结束；
- 专家姓名；
- 本次会诊时长；
- 已同步当前步骤与设备上下文；
- 提示后续可接入真实音视频和会诊纪要。

点击“返回检修向导”关闭工作台，原检修步骤、勾选状态和页面位置保持不变。

## 6. 视觉规范

- 风格：深色工业调度终端，克制、专业、适合油气场站软件。
- 主背景：深蓝黑；视频区：纯黑 `#000`。
- 主强调色：工业青蓝；连接成功：低饱和绿色；结束操作：警示红。
- 字体：继承现有系统字体，数字计时使用等宽数字特征。
- 动效：遮罩进入 180–240ms；连接扫描 1.6s 循环；状态切换 180ms。
- 不使用大面积渐变、人物头像或拟真视频纹理，避免将占位画面误认为真实接入。

## 7. 可访问性与容错

- 会诊工作台使用 `role="dialog"` 与 `aria-modal="true"`。
- `Esc`：选择页直接关闭；会诊中打开结束确认；结束确认页取消确认。
- 所有图标按钮都有可读标签与 `title`。
- 连接计时器和自动状态定时器在组件卸载时清理。
- 支持 `prefers-reduced-motion`，关闭非必要动画。
- 窄屏下资料栏移至画面下方，操作栏允许换行。

## 8. 演示脚本（约 25 秒）

1. 在检修向导某一步点击“专家视频会诊”。
2. 指出系统已自动带入场站、设备、告警和当前检修步骤。
3. 保持推荐的王海峰专家，点击“发起视频会诊”。
4. 展示创建会诊室、同步上下文和专家接入过程。
5. 进入黑屏会诊主界面后说明：“这里已完成会诊流程和业务上下文同步，后续接入企业视频能力即可显示实时画面。”
6. 点击结束会诊并返回检修向导，证明主流程状态未丢失。

## 9. 验收标准

- 入口不再显示“预留”，点击后能够打开专家选择页。
- 选择、连接、会诊、结束、返回五段流程均可操作。
- 所有视频区域始终为黑屏，不触发浏览器权限请求。
- 会诊中能够看到当前检修步骤和关键事件信息。
- 关闭或结束后，检修向导当前步骤与勾选状态保持不变。
- 桌面宽屏和 760px 以下窄屏均无关键内容遮挡。
- 前端生产构建通过，控制台无新增运行时错误。

## 10. 后续真实接入边界

未来接入真实视频时，保留本版 UI 与业务状态机，将“连接中”和“会诊中”状态的数据源替换为企业音视频 SDK；会议号、参会状态、媒体开关和网络质量由 SDK 回调驱动。设备、告警和检修步骤仍由当前业务页面提供，不与视频服务耦合。
