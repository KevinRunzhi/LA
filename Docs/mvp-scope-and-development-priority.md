# MVP 功能清单与开发优先级：工控设备异常检测与故障分析系统

## 1. 文档目标

本文档用于冻结第一版 MVP 范围，目标是尽快做出一个能跑、能演示、能持续迭代的版本。

第一版不追求完整企业系统，不追求真实泛化能力，不追求复杂三维可视化。第一版只做一个设备、一个故障、一个完整闭环。

MVP 主演示场景：

```text
某输气场站站控柜内研华 ACP-4000 / IPC-610 工控机出现温度告警，
现场判断为风道堵塞 / 滤网积尘 / 散热异常方向，
系统引导一线人员按步骤完成检查、处理和恢复验证。
```

MVP 核心闭环：

```text
用户输入或触发预设异常
  -> 系统生成诊断摘要和步骤选项
  -> 用户进入步骤式检修向导
  -> 每一步展示预标注图片、操作说明、安全提醒和完成按钮
  -> 用户完成风道 / 滤网 / 风扇 / 恢复验证步骤
  -> 系统生成检修记录
  -> 专家修正一条经验
  -> 修正内容写入知识库 / 图谱 / 专家经验
  -> 再次触发同类异常时命中专家修正
```

首期深度演示设备：

| 设备 | 范围 |
|---|---|
| 研华 ACP-4000 / IPC-610 工控机整机 | 第一版唯一主演示设备 |

首期主演示故障：

| 设备 | 故障 |
|---|---|
| 研华 ACP-4000 / IPC-610 工控机整机 | 高温告警 / 风道堵塞 / 散热异常 |

暂不进入第一版主演示：

1. PLC 控制柜完整诊断。
2. 通信交换机、传感器、现场仪表、电动执行机构。
3. 供电异常、硬盘异常、系统死机等其他工控机故障。
4. 真实视频识别、真实语音播报、真实连线专家。

---

## 2. 开发优先级定义

### P0：必须完成

P0 是第一版演示闭环必需功能。缺少任何一项，MVP 就不完整。

### P1：建议完成

P1 能增强演示可信度，但如果时间紧，可以静态化或延后。

### P2：后期增强

P2 不影响第一版演示。只在 P0 稳定后考虑。

## 3. P0 必做功能

### 3.1 龙芯空壳运行验证

目标：

- 尽早解决龙芯兼容性问题。
- 不等所有功能做完再上龙芯。

功能：

1. 前端空壳页面能在龙芯上通过 Node/Vite 开发态启动。
2. Flask Hello API 能在龙芯上启动。
3. 前端能请求后端 Hello API。
4. 能展示一个空的工作台页面。
5. 记录 Node、npm、Python、Flask 的版本和启动命令。

验收：

```text
龙芯本机打开前端页面
  -> 页面显示系统名称
  -> 点击测试按钮
  -> 后端返回 hello
```

不做：

- 平板访问龙芯 IP。
- Nginx 正式部署。
- Docker。
- 云 API 龙芯实时联调。

---

### 3.2 基础页面骨架

目标：

- 先做出可演示的页面结构。
- 为后续功能填充留入口。

页面：

1. 工作台。
2. 异常检测入口。
3. 步骤式检修向导。
4. 检修记录页面。
5. 专家审核与知识回流页面。

功能：

1. 左侧或顶部导航。
2. 当前用户信息，可先写死。
3. 当前主演示设备提示。
4. 当前主演示故障提示。
5. 一键进入主演示流程。

不做：

- 完整后台管理。
- 复杂权限。
- 多角色菜单控制。

---

### 3.3 异常检测入口

目标：

- 用户用一句话或点击预设异常进入分析阶段。
- 对话只作为入口，不作为主体。

功能：

1. 输入现场现象：

```text
站控柜内工控机温度告警，风扇声音异常，怀疑散热不良。
```

2. 提供“一键触发主演示异常”按钮。
3. 系统生成诊断摘要：
   - 设备：研华 ACP-4000 / IPC-610 工控机。
   - 故障：高温告警 / 风道堵塞 / 散热异常。
   - 风险：可能影响数据上传稳定性和监控告警可靠性。
4. 展示步骤选项：
   - 定位异常设备。
   - 查看故障分析。
   - 进入检修步骤。
   - 提交专家审核。

实现要求：

1. 第一版可以完全使用预设数据。
2. 可保留调用云 API 的接口，但必须有缓存兜底。
3. 大模型返回失败不能阻断演示。

---

### 3.4 步骤式检修向导

目标：

- 用步骤流替代大段文字回答。
- 用户按步骤执行，每一步都有图片、说明、安全提醒和完成按钮。

主演示步骤：

| 步骤 | 名称 | 展示图片 | 完成条件 |
|---|---|---|---|
| 1 | 定位异常设备 | `step-01-location.png` | 用户确认发生告警的站控柜和工控机 |
| 2 | 确认安全条件 | `step-02-safety.png` | 用户确认通知值班负责人、静电防护、禁止带电拆卸 |
| 3 | 检查风道 | `step-03-airflow.png` | 用户确认进风口、出风口和机柜通风情况 |
| 4 | 检查滤网和风扇 | `step-04-filter-fan.png` | 用户确认滤网积尘、风扇异响和运行状态 |
| 5 | 恢复验证 | `step-05-verify.png` | 用户确认温度告警解除、数据上传稳定 |

功能：

1. 步骤列表。
2. 当前步骤详情。
3. 当前步骤预标注图片展示。
4. 操作说明。
5. 安全提醒。
6. 来源依据简要展示。
7. 上一步 / 下一步。
8. 标记完成。
9. 步骤完成状态。

实现要求：

1. 图片由团队手动标注后放入素材目录。
2. 前端只按步骤显示对应图片。
3. 不做热点识别。
4. 不做真实三维。
5. 不做可旋转模型。

---

### 3.5 检修记录生成

目标：

- 向导完成后形成一次检修记录。
- 作业卡不作为核心演示，只作为记录导出结果。

功能：

1. 汇总用户执行过的步骤。
2. 汇总每一步完成状态。
3. 汇总诊断摘要、风险、影响范围、安全检查结果。
4. 展示检修完成记录。
5. 支持打开打印版页面。
6. 支持浏览器另存为 PDF。

不做：

- 后端 PDF 生成。
- Word 导出。
- 复杂排版。

---

### 3.6 轻量 RAG 证据与知识来源

目标：

- 让关键诊断和步骤有依据。
- 不做复杂文档解析。

功能：

1. 预置 5-10 条知识条目。
2. 针对高温、风道、滤网、风扇、安全要求返回固定证据卡片。
3. 每条证据展示：
   - 来源名称。
   - 章节或条目名。
   - 简短引用。
   - 关联步骤。

实现要求：

1. 第一版可使用 SQLite 关键词匹配或直接按步骤绑定证据。
2. 不接 DashVector。
3. 不做 PDF 导入。
4. 不做扫描件 OCR。

---

### 3.7 知识图谱子图

目标：

- 保留赛题要求中的知识图谱能力。
- 展示当前故障相关关系，不追求大规模图谱。

功能：

1. 展示当前故障相关子图。
2. 包含设备、部件、故障、原因、检查方法、处理措施、安全要求、影响范围。
3. 专家修正后可新增或更新 1 条关系。

主演示子图：

```text
某输气场站 -> 包含 -> 站控柜
站控柜 -> 包含 -> 工控机
工控机 -> 包含 -> 风道
工控机 -> 包含 -> 滤网
工控机 -> 包含 -> 风扇
工控机 -> 发生 -> 高温告警
高温告警 -> 可能原因 -> 风道堵塞
高温告警 -> 可能原因 -> 滤网积尘
高温告警 -> 可能原因 -> 风扇异常
高温告警 -> 影响 -> 数据上传稳定性
高温告警 -> 影响 -> 监控告警可靠性
处理措施 -> 前置安全要求 -> 静电防护
处理措施 -> 前置安全要求 -> 禁止带电拆卸非热插拔部件
```

技术：

1. SQLite 存 `kg_entities` 和 `kg_relations`。
2. 前端用 ECharts graph 或简单 SVG 展示。
3. 不使用 Neo4j。
4. 不做图算法。

---

### 3.8 专家审核与回流

目标：

- 展示系统“越用越准”。
- 只做一条专家修正闭环。

功能：

1. 待审核列表展示当前检修记录。
2. 专家打开记录，查看：
   - 用户输入。
   - 诊断摘要。
   - 步骤执行结果。
   - 证据卡片。
   - 知识图谱关系。
3. 专家填写修正文本：

```text
沙尘环境下，站控柜工控机滤网维护周期应从季度检查缩短为月度检查。
恢复运行后需观察数据上传稳定不少于 10 分钟。
```

4. 专家选择回流目标：
   - 写入知识库。
   - 更新图谱关系。
   - 写入专家经验。
5. 审核通过后，再次触发同类异常时展示：

```text
专家修正 · 已审核
```

不做：

- 两步复杂审核。
- 多字段结构化编辑。
- 驳回/退回补充的复杂状态。

---

### 3.9 演示兜底

目标：

- 不让云 API、网络、图片缺失影响演示。

功能：

1. 预置诊断摘要。
2. 预置步骤选项。
3. 预置 Agent 会诊结果。
4. 预置证据卡片。
5. 预置知识图谱。
6. 预置专家修正演示数据。
7. 一键重置演示数据。

## 8. 演示主流程

主演示场景：

```text
站控柜内研华 ACP-4000 / IPC-610 工控机温度告警，
风扇声音异常，怀疑风道堵塞或滤网积尘导致散热异常。
```

演示步骤：

1. 打开系统工作台。
2. 点击“开始异常检测”。
3. 输入现场现象，或点击“一键触发主演示异常”。
4. 系统生成诊断摘要和步骤选项。
5. 展示多 Agent 会诊结果。
6. 点击“进入检修向导”。
7. 第 1 步：显示异常设备定位图片。
8. 第 2 步：显示安全确认图片和安全提醒。
9. 第 3 步：显示风道检查图片。
10. 第 4 步：显示滤网和风扇检查图片。
11. 第 5 步：显示恢复验证图片。
12. 生成检修记录。
13. 切换到专家审核页面。
14. 专家补充“沙尘环境滤网维护周期缩短”等经验。
15. 审核通过并写入知识库、图谱和专家经验。
16. 再次触发同类异常。
17. 系统展示“专家修正 · 已审核”内容。

## 9. 开发顺序建议

推荐按“能跑起来优先”开发：

1. 龙芯空壳验证：前端 Hello + Flask Hello。
2. 页面骨架：工作台、异常检测入口、向导页。
3. 演示种子数据：步骤流、图片路径、证据、图谱。
4. 步骤式检修向导：步骤切换、图片展示、完成状态。
5. 检修记录生成。
6. 专家审核与回流。
7. 再次触发命中专家修正。
8. 知识图谱子图展示。
9. 多 Agent 会诊卡片和过程动画。
10. 打印页导出。
11. 一键重置演示数据。

---

## 10. 验收标准

MVP 完成标准：

1. WSL 环境能完整跑通主演示流程。
2. 龙芯环境能启动前端 dev server 和 Flask 后端。
3. 工作台能进入异常检测入口。
4. 系统能生成诊断摘要和步骤选项。
5. 系统能进入步骤式检修向导。
6. 每个步骤能显示对应预标注图片。
7. 用户能逐步完成 5 个步骤。
8. 系统能生成检修记录。
9. 系统能展示轻量证据卡片。
10. 系统能展示当前故障知识图谱子图。
11. 专家能填写一条修正并审核通过。
12. 修正内容能写入知识库 / 图谱 / 专家经验。
13. 再次触发同类异常时能展示“专家修正 · 已审核”。
14. 云 API 不可用时，主演示流程仍可通过缓存走通。
15. 全流程保持只读不控，不接入真实生产控制网络。
