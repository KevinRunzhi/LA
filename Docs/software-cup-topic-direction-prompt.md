## 2. 官方赛题信息

赛题名称：

> A1 - 基于多模态大模型技术的设备检修知识检索与作业系统

组类：

> A组（本科、研究生、高职）

出题企业：

> 龙芯中科技术股份有限公司

赛题简介：

本赛题聚焦工业、能源、制造等领域设备检修的核心痛点，以多模态大模型技术为核心支撑，引导参赛者开发一款兼具智能化、实用性与创新性的设备检修知识检索与作业辅助系统。赛题旨在推动大模型技术与行业场景深度融合，破解传统设备检修中知识分散、检索低效、作业不规范、经验传承难等问题，打造可落地、高价值的行业智能助手，助力行业数字化转型与效率升级。

官方业务场景：

在钢铁、汽车制造等行业，大型企业产线设备密集且联动性强，设备稳定运行直接决定产能与效益。企业在传统运维中存在检修知识分散于手册、日志与老员工经验中形成“数据孤岛”、新人上手慢、非计划停机单次损失超百万元等痛点，亟需智能工具破解经验传承难、故障响应慢的问题。

基于上述业务背景，聚焦制造业核心生产设备检修，参赛者可基于本地大模型或 API（如 Qwen、DeepSeek 等），开发适配现场检修作业的系统，整合多模态检修资源，辅助一线人员快速检索方案、诊断故障，实现知识数字化沉淀与高效复用。

官方说明中提到“检修设备不限，可自行发挥”，因此我们可以选择更适合比赛演示的细分设备场景。

## 3. 官方基本功能需求

请严格围绕以下功能要求分析方案，不要偏离赛题。

### 3.1 部署与界面

采用 B/S 架构开发，软件需部署在自主指令系统 LoongArch 架构 + 银河麒麟高级服务器版上运行。

注意：

> 不满足该要求视为 0 分。

系统需支持本地部署大模型服务或云端大模型服务，具备 PC Web 或 App 可视化界面，方便用户访问与操作。

### 3.2 多模态知识检索

系统需支持：

- 文本输入
- 故障图片输入
- 设备型号输入

并实现：

- 精准语义检索
- 跨模态匹配
- 快速调取检修手册等资源

### 3.3 标准化作业指引

系统需嵌入检修全流程，提供：

- 步骤化操作指引
- 合规校验提醒
- 根据设备类型推送流程
- 根据检修等级个性化推送流程

目标是助力规范作业、降低操作失误率。

### 3.4 知识沉淀与更新

系统需支持：

- 一线人员上传检修案例
- 一线人员上传经验总结
- 内容经审核后纳入知识图谱
- 知识实时更新
- 手动标注与修正大模型输出结果
- 优化系统适配性

## 7. 技术与部署约束

请在分析时考虑以下现实条件：

1. 时间短，必须优先保证主流程跑通。
2. 龙芯 LoongArch 平台生态弱于 x86/ARM，部分 Python、Node、AI 依赖可能难部署。
3. 不建议在龙芯本地部署大模型、向量数据库、图数据库或重型 AI 框架。
4. 数据库、向量库、大模型可以使用云服务。
5. 本地应用应尽量轻量，例如：
   - Web 前端
   - Python/Node 后端
   - Nginx
   - 简单配置和日志
6. 重型能力建议云化，例如：
   - 关系型数据库
   - 向量数据库
   - 多模态大模型
   - 文本生成模型
   - 图数据库，若确实需要

请不要把方案设计成依赖本地训练模型、本地部署大模型或复杂 Docker 编排的重型系统。

## 8. 赛题本质理解要求

请先分析该赛题到底要解决什么问题。

不要只把它理解成普通聊天机器人，也不要只理解成普通知识库搜索系统。

我们认为它应该面向真实企业中的设备检修场景，解决以下问题：

1. 设备手册、检修规程、历史工单、专家经验分散。
2. 一线检修人员遇到故障时，很难快速找到准确处理方法。
3. 新员工依赖老师傅经验，知识传承困难。
4. 现场故障往往包含文本、图片、仪表读数、报警代码等多模态信息。
5. 大模型直接回答可能不可靠，需要结合企业知识库和专家修正。
6. 检修不仅要知道原因，还要生成标准化作业步骤和安全注意事项。
7. 高质量检修经验应该能沉淀回知识库。

请基于这些点，重新解释赛题本质，并说明这个赛题真正考察的能力是什么。

- 开关柜

3. 使用 RAG 生成故障原因分析和检修建议。
4. 通过知识图谱展示设备、部件、故障、原因、检测方法、处理措施之间的关系。
5. 生成标准化检修作业卡。
