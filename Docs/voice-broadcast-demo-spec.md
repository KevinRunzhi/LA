# 语音播报胶囊演示功能 Spec

> 版本：V1.0
> 日期：2026-07-15
> 范围：工程师端检修向导中的语音播报演示；纯前端动画，不输出真实语音。

## 1. 背景

检修向导底部现有“语音播报”按钮仅显示“预留”标记，无法在汇报中表达智能体将当前检修内容组织为语音并播报的过程。本功能以用户提供的深色语音胶囊为视觉参考，补齐“Agent 生成播报内容”和“声纹播报”两个连续阶段。

## 2. 目标

- 点击“语音播报”后展示完整、可重复的演示流程。
- 在声纹出现前先播放约 4.5 秒的 Agent 生成动画。
- 声纹阶段由演示人员手动结束，方便现场讲解停留。
- 左侧取消、右侧完成的结构与参考图保持一致。
- 不播放音频、不调用语音接口、不请求浏览器权限。
- 结束或取消后不改变检修向导的当前步骤和检查进度。

## 3. 非目标

- 不进行文字转语音或语音合成。
- 不加载预制音频文件。
- 不接入讯飞、浏览器 SpeechSynthesis 或其他音频 SDK。
- 不实现音量、语速、音色和播放进度控制。
- 不保存播报记录。

## 4. 入口

- 位置：检修向导底部操作区，位于“专家视频会诊”之后。
- 原状态：`语音播报 · 预留`。
- 新状态：`语音播报`，移除“预留”标记。
- 图标：沿用现有扬声器图标。
- 点击结果：在页面底部中央弹出语音播报胶囊，不打开全屏遮罩。

## 5. 状态机

| 状态 | 持续时间 | 画面 | 用户操作 |
|---|---:|---|---|
| `closed` 关闭 | 持续 | 普通语音播报按钮 | 点击进入生成中 |
| `generating` Agent 生成中 | 4.5 秒 | 左侧取消、中央生成动画、右侧留空 | 可取消 |
| `broadcasting` 播报中 | 手动结束 | 左侧取消、中央动态声纹、右侧完成 | 可取消或完成 |
| `completing` 完成反馈 | 600 毫秒 | 声纹收束、绿色对勾、播报完成 | 自动关闭 |

状态流：

```text
closed → generating → broadcasting → completing → closed
              └──────── 取消 ───────────────→ closed
```

## 6. Agent 生成阶段

### 6.1 时长与文案

生成阶段总时长为 4.5 秒，主文案始终为“Agent 生成中”，尾部三个点循环变化。辅助文案按以下节奏切换：

| 时间 | 辅助文案 |
|---:|---|
| 0–1.5 秒 | 正在提取当前检修步骤 |
| 1.5–3.0 秒 | 正在组织安全提示与操作顺序 |
| 3.0–4.5 秒 | 语音播报内容已就绪 |

### 6.2 视觉

- 胶囊与播报阶段使用同一尺寸，避免状态切换时整体跳动。
- 左侧显示圆形取消按钮。
- 中央显示青蓝色旋转光环、主文案和辅助文案。
- 右侧完成按钮不显示，但保留等宽布局空间，保证中央内容稳定。
- 胶囊上沿显示一条细小进度光带，在 4.5 秒内从左至右完成。

### 6.3 进入播报

生成结束后使用约 400 毫秒过渡：

- 生成文字向中央收拢并淡出；
- 旋转光环淡出；
- 声纹由中央向两侧展开；
- 右侧完成按钮弹入；
- 辅助提示切换为“正在播报当前检修步骤”。

## 7. 声纹播报阶段

### 7.1 结构

胶囊从左到右分为三部分：

1. 取消按钮：深灰圆形底、白色叉号；
2. 声纹区：约 19 根竖向音频柱，中央较高、两侧较低；
3. 完成按钮：白色圆形底、深色对勾。

### 7.2 声纹运动

- 声纹持续循环，不自动结束。
- 各音频柱使用不同相位和速度，避免同步机械升降。
- 中央音频柱振幅最大，边缘振幅逐渐降低。
- 颜色以白色为主，中央少量使用工业青蓝强调色。
- 脚本只根据时间绘制抽象声纹，不读取麦克风或音频数据。

### 7.3 操作

- 点击左侧取消：立即收起胶囊，恢复普通按钮，不显示完成反馈。
- 点击右侧完成：进入完成反馈。
- 胶囊出现期间再次点击底部“语音播报”按钮，行为等同于取消。

## 8. 完成反馈

- 声纹在约 180 毫秒内向中央收束。
- 右侧对勾由白色切换为低饱和绿色。
- 中央显示“播报完成”。
- 反馈总时长约 600 毫秒，随后胶囊自动收起。

## 9. 视觉规范

- 参考尺寸：桌面端约 `310 × 76px`，窄屏约 `280 × 70px`。
- 位置：视口底部中央，距离底部约 88px，避开检修向导操作栏。
- 背景：接近纯黑的半透明深色背景。
- 边框：双层深灰描边，保留参考图的实体胶囊感。
- 圆角：胶囊半径约为高度的一半。
- 阴影：使用克制的黑色投影和一圈低亮青蓝外发光。
- 字体：继承现有系统字体，主文案为中等字重，辅助文案更小更淡。
- 层级：高于业务内容，低于专家视频会诊全屏层。

## 10. 响应式规则

- 桌面端固定在视口底部中央。
- 760px 以下宽度缩小胶囊、按钮和文字，不允许横向溢出。
- 当页面高度不足时，胶囊距离底部调整为 72px。
- 不遮挡浏览器底部安全区。

## 11. 可访问性

- 胶囊使用 `role="status"` 和 `aria-live="polite"` 宣告状态变化。
- 取消与完成按钮具有明确的 `aria-label` 和 `title`。
- `Escape` 在生成中或播报中均可取消。
- 支持 `prefers-reduced-motion`：旋转和波动降低为静态状态，但时间与交互流程保持一致。
- 所有按钮具有清晰的键盘焦点样式。

## 12. 技术方案

- 使用独立 React 组件管理状态和定时器。
- 使用 Portal 挂载到 `document.body`，避免被检修页面布局裁切。
- 使用 Canvas 绘制动态声纹，按固定算法生成确定性波形。
- 所有 `setTimeout`、`requestAnimationFrame` 和键盘监听在卸载时清理。
- 不新增第三方依赖。

## 13. 演示脚本

1. 在检修向导中点击“语音播报”。
2. 指出系统正在提取当前步骤、组织安全提示和操作顺序。
3. 等待约 4.5 秒，展示 Agent 生成动画自动切换为动态声纹。
4. 讲解：“这里是语音播报演示，后续可接入预制音频或企业语音服务。”
5. 点击右侧对勾，展示“播报完成”并自动收起。
6. 再次点击可重复演示；点击左侧叉号可直接取消。

## 14. 验收标准

- “语音播报”入口不再显示“预留”。
- 点击后先出现约 4.5 秒 Agent 生成动画。
- 三段辅助文案按顺序切换。
- 生成结束后自动进入持续声纹动画。
- 左侧取消和右侧完成均可用。
- 完成后显示短暂反馈并自动收起。
- 重复点击可以重新播放完整流程。
- 不产生真实声音，不请求媒体权限。
- 返回检修向导时当前步骤和检查进度不变。
- 生产构建通过，浏览器控制台无新增错误。
