# 专家知识图谱展示 Spec

> 版本：V1.0
> 日期：2026-07-12
> 定位：案例回流演示闭环中的专家知识图谱展示页

## 1. 目标

让专家和演示观众明确看到：专家修订的案例经验已经转化为知识节点与关系，正式进入知识图谱，并可追溯到来源案例和知识版本。

第一版以录制展示为主，不建设通用图数据库，不在图谱总览页提供编辑入口。关系编辑仍统一发生在专家审核第二步。

## 2. 入口与信息架构

专家左侧菜单固定为：专家工作台、全部案例、检修知识库、知识图谱。

知识图谱页面由状态条、图谱画布、视图切换和节点详情组成。

## 3. 两种视图

### 3.1 当前图谱

展示设备、异常告警、监测参数、故障原因、检修动作、恢复标准、来源案例之间的完整关系。

### 3.2 本次更新

- 降低既有节点透明度；
- 突出本次新增或修改的节点；
- 发布前显示橙色虚线和“待发布”；
- 发布后显示绿色实线和“专家已确认”；
- 显示来源案例 `CASE-ACP4000-001` 和知识版本 `KB-008 V1.0 → V1.1`。

## 4. 节点类型与颜色

| 类型 | 示例 | 颜色 |
|---|---|---|
| 设备 | 研华 ACP-4000 / IPC-610 | 蓝色 |
| 异常告警 | TEMP/FAN 告警 | 红色 |
| 监测参数 | 风扇转速低于 500 rpm | 灰蓝色 |
| 故障原因 | 滤网积尘、风扇老化 | 橙色 |
| 检修动作 | 清理滤网、更换风扇 | 青色 |
| 恢复标准 | 告警解除、转速恢复 | 绿色 |
| 来源案例 | CASE-ACP4000-001 | 紫色 |

## 5. 节点详情

点击节点后显示节点名称与类型、适用设备、知识版本、来源案例、专家状态、关联关系和发布状态。

## 6. 状态联动

- `knowledgePublished = false`：新增关系为审核草稿，提供“进入专家审核”；
- `knowledgePublished = true`：显示 KB-008 V1.1 已写入图谱；
- 新关系绑定来源 `CASE-ACP4000-001`；
- 演示重置后恢复发布前状态。

### 6.1 唯一关系数据源

专家审核第二步编辑的 `relations` 在发布后写入 `presentation_state.json` 的 `publishedRelations`。发布结果页、知识图谱“本次更新”、知识版本差异接口和知识应用验证必须统一读取 `publishedRelations`，禁止继续使用案例文件中的预设 `graphChanges` 代替专家实际修改。只有尚未产生草稿或正式发布关系时，才允许用 `graphChanges` 作为初始化模板。

## 7. 演示路径

工程师提交案例 → 专家三步审核 → 发布 KB-008 V1.1 → 左侧进入知识图谱 → 选择“本次更新” → 点击“风扇老化”或来源案例节点 → 展示新增关系、专家确认和案例追溯。

## 8. 验收标准

- 左侧存在独立知识图谱入口；
- 当前图谱和本次更新可以切换；
- 节点可以点击并更新详情；
- 发布前后状态有明显差异；
- 新增关系能够追溯到 CASE-ACP4000-001 和 KB-008 V1.1；
- 1920×1080 下不重叠、不溢出；
- 浏览器控制台无错误，Node 20.19.4 构建通过。
