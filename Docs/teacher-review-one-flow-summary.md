# 老师评审单流程摘要

更新时间：2026-07-10

本文用于现场走查，只保留一条最能解释产品价值的流程。

## 当前 R0 走查

```text
首页输入异常
  -> 单页动态接诊工作区
  -> 中央位置图持续显示
  -> 接诊 Agent 自动流式分析
  -> 设备信息卡动态生成
  -> 工程师确认设备
  -> 告警状态卡动态生成
  -> 工程师修改并确认状态
  -> 接入摘要最后生成
  -> 触发诊断
  -> 进入现有多 Agent 会诊
```

讲解重点：

- 页面不是提前摊开的固定表单；任务由当前处理阶段逐步生成。
- Agent 展示的是输入、检索依据、结果和下一任务，不展示隐藏思维链。
- 工程师始终拥有最终确认权。
- 修改字段会反映到摘要中，证明不是纯视频或纯动画。

## 完整 MVP 目标流程

```text
动态接诊
 -> 分析诊断
 -> 可编辑预方案
 -> 步骤式检修向导
 -> 恢复验证
 -> 检修记录
 -> 上传总结
 -> 专家审核归档
 -> 知识图谱新增知识
```

当前只把 R0 走通后再进入后续模块，不把未完成能力包装成已经实现。

## 演示环境

- WSL 与龙芯统一 Node.js `20.19.4`。
- 龙芯使用 `npm run dev` 开发态逐模块验证。
- 主流程在龙芯演示；pad 后续负责语音等移动端多模态效果。
- 模拟能力必须在讲解中明确说明数据来自人工审定案例。
