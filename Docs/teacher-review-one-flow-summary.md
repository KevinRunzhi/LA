# 站控慧眼：操作流程与功能说明

## 1. 系统能做什么

本系统面向油气管道场站工控设备检修场景，首期聚焦 **研华 ACP-4000 / IPC-610 工控机整机**，帮助一线巡检与工控维护人员完成：

1. 描述现场故障现象。
2. 上传工控机面板照片、机柜环境照片或告警截图。
3. 自动检索工控机维护知识、巡检规程和历史案例。
4. 通过多 Agent 协同给出检修建议。
5. 展示故障原因、检查方法、处理措施和影响范围之间的知识图谱。
6. 生成可执行的标准化检修作业卡。
7. 由专家审核和修正结果。
8. 将专家经验回流到知识库，后续同类问题优先引用。

简单来说：

> 巡检员把工控机现场问题告诉系统，系统给出有依据的检修建议和作业卡；专家修正后，系统下次回答得更准。

PLC 控制柜、SCADA 监控平台、通信链路和现场监测点暂作为关联对象和影响范围节点保留，不进入首期主诊断闭环。

---

## 2. 一条完整操作流程

### 第一步：一线人员登录系统

一线巡检人员使用账号登录，进入工作台。

工作台提供：

1. 开始智能诊断入口。
2. 最近诊断记录。
3. 专家待审核数量提示。
4. 当前主演示设备：研华 ACP-4000 / IPC-610 工控机整机。

体现功能：

- 基础登录与角色区分。
- PC Web 可视化操作界面。

---

### 第二步：描述现场故障

用户进入“智能诊断台”，直接输入现场现象。

示例：

```text
站控柜内工控机温度告警，风扇声音异常。
```

系统继续追问：

1. 场站位置是什么？
2. 机柜位置是什么？
3. 工控机型号是什么？
4. 检修等级是什么？
5. 是否有面板照片、机柜环境照片或告警截图？

体现功能：

- 智能接诊。
- 不要求巡检人员一开始就知道准确故障名称。
- 适合新员工和现场快速使用。

---

### 第三步：上传图片作为辅助信息

用户上传工控机面板照片、机柜环境照片或告警截图。

系统识别图片中的关键信息，例如：

1. 疑似设备类型。
2. 面板或指示灯状态。
3. 风扇、滤网、进风口等可见异常。
4. 机柜环境风险，例如积尘、通风不畅。
5. 需要人工确认的内容。

体现功能：

- 多模态输入。
- 图片辅助识别。
- 识别不确定时允许人工确认，避免误判。

---

### 第四步：系统检索知识并给出建议

系统根据用户描述、设备型号、图片识别结果和检修等级，自动检索知识库。

系统输出：

1. 可能故障类型：高温/散热异常。
2. 可能原因：滤网堵塞、风扇故障、风道不畅、机柜环境温度过高。
3. 推荐检查方法：查看风扇、检查滤网、检查进出风通道、核对告警记录。
4. 初步处理建议：按规程停机或切换后清洁滤网、检查风扇、复测温度。
5. 安全注意事项：断电、静电防护、禁止带电拆装、恢复后确认数据上传稳定。
6. 相似历史案例。

每条建议都显示来源，例如：

1. 来源文档。
2. 页码或章节。
3. 引用片段。
4. 相关案例。

体现功能：

- RAG 知识检索。
- 证据卡片。
- 回答有依据，不是普通聊天式回答。

---

### 第五步：多 Agent 协同会诊

系统组织多个 Agent 角色共同分析。

首期 Agent 包括：

| Agent | 作用 |
|---|---|
| 接诊 Agent | 整理用户描述，补齐场站、设备、型号、等级等信息 |
| 知识检索 Agent | 检索手册、规程、案例和图谱证据 |
| 工控设备专家 Agent | 从工控机整机检修角度分析故障原因和检查顺序 |
| PLC 系统专家 Agent | 当问题涉及 PLC、数据上传或影响范围时参与分析 |
| 安全合规 Agent | 检查断电、静电防护、禁止热插拔、恢复确认等安全要求 |

系统最终汇总为统一建议。

体现功能：

- 多 Agent 协同。
- 老师傅经验数字化。
- 技术判断和安全合规同时参与。

---

### 第六步：查看知识图谱和影响范围

系统展示当前故障相关的知识图谱。

示例路径：

```text
工控机
  -> 散热系统
  -> 高温/散热异常
  -> 滤网堵塞 / 风扇故障 / 风道不畅
  -> 清洁滤网 / 检查风扇 / 检查机柜通风
  -> 影响范围：监控告警可靠性、数据上传稳定性、历史记录完整性
```

用户可以点击图谱节点查看说明和来源。

体现功能：

- 知识图谱。
- 故障原因链路可解释。
- 影响范围可视化，不只是返回一段文字。

---

### 第七步：生成标准化作业卡

用户点击“生成作业卡”。

系统生成一张标准化检修作业卡，内容包括：

1. 场站位置和机柜位置。
2. 设备型号。
3. 故障现象。
4. 检修等级。
5. 风险等级。
6. 影响范围。
7. 作业前安全检查。
8. 操作步骤。
9. 恢复运行确认条件。
10. 验收标准。
11. 异常升级条件。
12. 知识来源。
13. 参与会诊的 Agent。

作业卡支持通过浏览器保存为 PDF。

体现功能：

- 标准化作业指引。
- 安全检查前置。
- 输出可执行结果，而不是只给建议。
- 支持 PDF 交付。

---

### 第八步：专家审核与修正

专家审核员登录系统，查看本次诊断结果。

专家可以修正：

1. 故障原因。
2. 检查方法。
3. 处理措施。
4. 作业卡步骤。
5. 知识图谱关系。
6. Agent 会诊意见。

示例修正：

```text
沙尘环境下，站控柜工控机滤网维护周期应从季度检查缩短为月度检查。
```

审核通过后，修正内容进入系统知识库。

体现功能：

- 专家审核。
- 人工修正大模型输出。
- 保证工业检修场景的可靠性。

---

### 第九步：知识回流，下一次回答更准

专家修正通过后，系统将内容写入：

1. 知识库。
2. 知识图谱。
3. 专家 Agent 经验记录。

当下一次遇到类似故障时，系统会优先引用专家已审核经验，并显示：

```text
专家修正 · 已审核
```

体现功能：

- 知识沉淀与更新。
- 经验传承。
- 系统越用越准。

---

## 3. 演示时可以这样讲

本系统演示一条完整闭环：

```text
现场现象输入
  -> 图片辅助识别
  -> 知识库检索
  -> 多 Agent 会诊
  -> 知识图谱与影响范围
  -> 生成 PDF 作业卡
  -> 专家审核修正
  -> 知识回流
```

核心价值：

1. 一线人员不用翻大量手册。
2. 新员工不知道准确关键词也能发起诊断。
3. 每条建议都有知识来源。
4. 作业卡能直接指导检修。
5. 安全要求不会被遗漏。
6. 老师傅经验可以沉淀到系统中。
7. 系统只读不控，不接入真实生产控制网络。
8. 系统部署在龙芯 + 银河麒麟环境，智能能力通过云端服务实现。

---

## 4. 首期演示范围

首期重点演示一个设备：

1. 研华 ACP-4000 / IPC-610 工控机整机。

重点演示故障：

1. 高温/散热异常。
2. 风扇故障/滤网堵塞/风道不畅。
3. 供电异常/冗余电源单路异常。
4. 硬盘故障/存储异常。
5. 指示灯/蜂鸣器/监控告警异常。
6. 系统运行异常/死机/数据上传中断。

PLC 控制柜、SCADA 监控平台、通信链路和现场监测点后续可通过配置知识库继续扩展。

---

## 5. 一句话总结

> 这是一个面向油气管道场站工控设备检修的智能助手：巡检人员描述工控机现场问题并上传照片，系统通过知识库、多 Agent、知识图谱和专家回流机制，生成有依据、可执行、可持续更新的检修作业方案。
