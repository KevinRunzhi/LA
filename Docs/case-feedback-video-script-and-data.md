## 1. 演示目标

视频需要证明以下完整闭环：

> 系统完成一次异常接入、诊断与检修后，自动生成真实案例；工程师确认实际结果并提交专家；专家修正关键结论并接受一条知识关系；系统发布知识新版本；新知识和图谱关系随后能够被下一次相似诊断命中。

视频不追求展示后台所有功能。历史案例、人员管理和预置知识主要服务 PPT，不进入视频主流程。

## 4. 分镜脚本

## 镜头 01：系统与场景

**时间：00:00–00:20**

页面：智能诊断入口。

画面必须出现：

- 李师傅 · 一线检修人员；
- 某输气场站 · 站控区域；
- 站控柜 A01 · 工控机；
- 温度告警或风扇异常快速接诊入口。

操作：

1. 简短停留系统首页；
2. 点击“温度告警”快速接诊。

解说重点：

> 本次演示处理站控柜 A01 内工控机的温度和风扇告警，并展示检修经验如何通过专家审核回流知识库。

禁止内容：

- 不展开人员管理；
- 不逐项介绍首页全部入口；
- 不展示历史案例。

冻结数据：

| 项目 | 值 |

| 持续时长 | 约 10 分钟 |

## 镜头 03：多 Agent 会诊

**时间：01:25–01:55**

页面：中央轻量会诊状态 + 右侧多 Agent 会诊消息流。

必须出现：

- 分析诊断 Agent；
- 知识检索 Agent；
- 操作合规 Agent；
- 右侧自动滚动；
- 输入、思考、依据和阶段结论；
- 三路证据汇合。

操作：不点击，只让动画运行。

解说重点：

> 三个 Agent 分别分析故障机理、检索维修知识和校验安全要求，会诊过程在右侧自动展开，中央保持业务状态。

录制要求：

- 至少完整录到一个 Agent 的思考和依据出现；
- 可以通过后期加速压缩等待时间；
- 不手动拖动右侧滚动条。

## 镜头 04：诊断结论

**时间：01:55–02:25**

页面：多 Agent 会诊与诊断结论。

冻结结论：

> 风道受阻、滤网积尘或风扇低速共同导致散热能力下降，其中风扇低速和温升数据相互印证，应优先检查滤网、风道和风扇模块。

必须出现：

- 诊断方向：工控机散热异常；
- 高优先级原因；
- TEMP/FAN 和风扇 `< 500 rpm` 依据；
- 操作安全边界；
- 进入检修预方案入口。

知识图谱第一次出现：

```text
工控机
→ TEMP/FAN 告警
→ 风扇转速偏低
→ 散热能力下降
→ 风道 / 滤网 / 风扇检查
```

图谱仅停留约 5–8 秒，用于说明诊断不是黑盒。

解说重点：

## 镜头 06：检修向导与恢复验证

**时间：02:55–03:30**

页面：步骤式检修向导。

必须完成的关键动作：

- 断电、挂牌和防静电确认；
- 风道与滤网检查；
- 风扇及接线检查；
- 恢复上电；
- 连续观察和告警解除确认。

实际结果冻结为：

- 滤网存在明显积尘；
- 清理滤网后温度下降；
- 原风扇转速仍偏低；
- 更换老化风扇后转速恢复；
- TEMP/FAN 告警解除；
- 连续运行 15 分钟正常。

视频操作：使用快速确认或预置演示结果，不逐项输入长文字。

1. 修改或一键采用上述专家结论；

## 7. 必须联动的页面变化

工程师提交后：

- 案例状态变为待专家审核；
- 专家工作台待审核数量增加；
- 专家账号能够打开案例；
- 工程师不能继续编辑已提交版本。

专家发布后：

- 案例状态变为已归档并沉淀知识；
- 待专家审核数量减少；
- `KB-008` 版本变为 V1.1；
- `KB-008` 增加来源案例；
- 图谱新增关系变为正式关系；
- 知识更新时间和修改专家发生变化。

验证知识后：

- 返回命中 `KB-008 V1.1`；
- 显示来源案例；
- 显示“风扇老化可能性较高”；
- 显示新增的推荐动作和图谱路径。

## 8. 视频不展示的数据

以下内容可以存在于 PPT 或后台展示数据中，但不得占用视频主线时间：

- 10–12 条历史摘要案例的逐条详情；
- 建设中第二案例的完整内容；
- 多专家会签；
- 退回补充完整分支；
- 管理员账号权限配置；
- 批量操作；
- 知识停用；
- PDF 导出和打印；
- 自由编辑大型知识图谱；
- 所有预置知识的逐条展示。
