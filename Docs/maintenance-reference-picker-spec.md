# 首页检修资料编组与异常接入引用 Spec

版本：V1.0
日期：2026-07-14
实施分支：`feature/maintenance-reference-picker`

## 1. 背景与目标

工程师首页“现场采集”目前支持图片、视频和音频，`检修资料`仍是不可用的预留入口。专家侧已经形成由历史案例、结构化检修知识和厂商原始手册组成的检修知识库，因此工程师接诊时应能从同一资料源中编组本次诊断参考资料。

本批次目标不是建设完整的资料上传、权限或持久化系统，而是完成一条可直接录制演示的前端闭环：

```text
首页点击“检修资料”
  → 打开检修资料编组台
  → 从案例 / 知识 / 手册中挑选
  → 确认形成“本次接诊资料包”
  → 首页入口显示已选状态
  → 资料包随异常接入持续保留
  → 第 5 步核对实际选中的资料
  → 最终摘要明确显示本次诊断引用资料
```

## 2. 产品边界

### 2.1 本批次实现

- 复用专家知识库的三类数据：历史检修案例、结构化检修知识、厂商手册。
- 首页提供检修资料编组台，不采用简单下拉菜单或树形菜单。
- 支持关键词搜索、按资料类型切换、推荐资料、选中/取消和清空。
- 选中结果以独立“诊断参考包”呈现，不与现场图片、视频、音频混合计数。
- 资料包在当前前端会话中持续保留，进入异常接入后不丢失。
- 异常接入顶部已知信息、第 5 步判断依据、最终摘要均显示真实选择结果。
- 后端不可用时提供演示兜底资料，保证页面仍可操作。
- 支持键盘 Escape 关闭、遮罩关闭、按钮焦点与基础响应式布局。

### 2.2 本批次不实现

- 不上传新的 PDF、Word 或案例文件。
- 不向后端写入“本次选择”，不做跨刷新、跨账号或跨设备持久化。
- 不解析文档内容，不把选中动作描述为真实 RAG 检索。
- 不改变专家知识库的数据编辑、审核和发布逻辑。
- 不自动替工程师选择资料；系统仅给出推荐，最终由工程师确认。
- 不新增第三方 UI、状态管理或动画依赖。

## 3. 核心概念与语义

| 概念 | 内容 | 生命周期 | 页面表达 |
| --- | --- | --- | --- |
| 现场证据 | 图片、视频、音频 | 当前接诊会话 | “现场材料 / 现场证据” |
| 诊断参考 | 案例、知识、手册 | 当前接诊会话 | “本次接诊资料包” |
| 推荐资料 | 根据主演示设备与故障方向预置的候选项 | 编组台打开期间 | “建议随本次接诊引用” |

两类资料必须分开展示。现场证据回答“现场发生了什么”，诊断参考回答“本次判断参考了什么”。

## 4. 交互与视觉方案

### 4.1 首页入口

“检修资料”卡从禁用态改为可点击态：

- 未选择：显示“从案例、知识与厂商手册中编组”，右侧动作显示“选择资料”。
- 已选择：卡片使用青绿色左侧状态条和轻微高亮，右侧显示“已选 N 项”。
- 现场证据区下方新增轻量“诊断参考包”停靠条，最多展示 3 个资料标签，其余以 `+N` 表示。
- 停靠条提供“调整资料”入口和单项移除动作。

### 4.2 检修资料编组台

采用接近全屏的工业资料工作台，而不是菜单：

```text
┌ 推荐带：当前对象 / 故障方向 / 推荐资料快捷选择 ┐
├ 类型标签 + 搜索 ─────────────────────────┤
│ 资料墙：案例卡 / 知识卡 / 手册卡      │ 本次资料包 │
│ 每张卡显示编号、类型、标题、适用对象    │ 已选数量   │
│ 故障领域或厂商标签、选中状态            │ 条目列表   │
│                                        │ 清空/确认  │
└────────────────────────────────────────┘
```

视觉方向：延续现有深色工业工作台，以“资料档案室 / 技术资料编组台”为记忆点。案例使用档案卡语义，知识使用检修知识卡语义，手册使用带竖向书脊的技术手册语义。选择时卡片产生短促上浮与描边点亮，右侧资料包同步出现条目。

### 4.3 选择规则

- 点击资料卡切换选中状态。
- 右侧资料包允许单项移除和全部清空。
- 推荐带只提供快速选择，不强制覆盖用户现有选择。
- 弹层内部使用草稿选择；只有点击“确认并用于本次接诊”才写入页面会话态。
- 关闭或点击遮罩视为取消本次未确认修改。
- 再次打开时，以当前已确认资料包作为草稿初值。

### 4.4 异常接入中的持续展示

- 第 1～5 步顶部“已知信息”增加“检修资料”条目，显示 `N 项 · X 案例 / Y 知识 / Z 手册`。
- 第 5 步将固定的两张判断依据卡替换为：
  - 当前运行阈值与告警标准卡；
  - 本次接诊资料包，展示已选资料标题和类型。
- 如果用户未选择资料，保留系统默认依据说明，并明确标注“未额外编组资料”。
- 最终摘要新增独立“诊断参考”区域，不挤占现场材料语义。

## 5. 数据设计

### 5.1 数据源

| 资料类型 | 接口 | 现有源文件 |
| --- | --- | --- |
| 案例 | `GET /api/admin/cases` | `backend/data/presentation/cases.json` |
| 知识 | `GET /api/admin/knowledge` | `backend/data/presentation/knowledge_base.json` |
| 手册 | `GET /api/admin/manuals` | `backend/data/presentation/manual_sources.json` |

三类接口也是专家知识库的现有数据源，本功能不建立第二套后端目录。

### 5.2 前端标准结构

```js
{
  key: "case:CASE-HIS-002",
  id: "CASE-HIS-002",
  type: "case", // case | knowledge | manual
  title: "站控工控机风扇异响排查",
  eyebrow: "历史案例",
  meta: "ACP-4000 工控机 · 风扇异常",
  tags: ["风扇异常", "工控机"],
  summary: "...",
  recommended: true
}
```

使用 `type:id` 作为稳定键，避免案例、知识和手册编号命名空间发生碰撞。

### 5.3 推荐规则

主演示路径固定推荐以下内容：

- `CASE-HIS-002`：站控工控机风扇异响排查；
- `KB-008`：风扇检查与更换；
- `DOC-ADVANTECH-ACP4000-IPC610-UM-ED6`：当前用户手册；
- `DOC-ADVANTECH-SAB2000-SM-ED2`：告警板启动手册。

本规则是演示推荐，不表述为真实相似度检索。

### 5.4 状态模型

```js
referenceCatalog     // 三类接口标准化后的完整资料目录
selectedReferences  // 已确认、贯穿当前接诊的资料包
referencePickerOpen // 编组台可见状态
```

编组台内部维护 `draftKeys`。确认后根据目录生成 `selectedReferences`；退出登录时清空，接诊步骤切换和诊断启动时保留。

## 6. 组件与代码落点

| 组件 / 逻辑 | 职责 | 建议位置 |
| --- | --- | --- |
| `maintenanceReferenceFallback` | 后端不可用时的演示资料 | `frontend/src/data/maintenanceReferenceCatalog.js` |
| `normalizeReferenceCatalog` | 将三类 API 结果统一为卡片模型 | 同上 |
| `ReferencePickerModal` | 搜索、类型标签、资料墙、选择草稿和确认 | `frontend/src/App.jsx`（本轮保持单体结构） |
| `ReferencePackageDock` | 首页资料包停靠条 | `frontend/src/App.jsx` |
| `ReferenceSummaryStrip` | 异常接入和摘要的紧凑展示 | `frontend/src/App.jsx` |
| 样式 | 编组台、卡片、选中态、动效、响应式 | `frontend/src/styles/app.css` |

当前 `App.jsx` 仍是项目既有单体入口。本批次优先最小改动，不顺带进行大规模组件目录重构。

## 7. 错误与降级

- 三类接口全部成功：展示完整目录和真实数量。
- 部分接口失败：成功类型使用接口数据，失败类型使用兜底数据。
- 全部接口失败：显示兜底目录，并在编组台标注“演示资料目录”。
- 搜索无结果：显示空状态与清除搜索入口。
- 目录刷新后找不到已选编号：保留当前已确认条目的快照，不静默删除。

## 8. 动效规范

- 弹层打开：遮罩 160ms 淡入，工作台 240ms 从下方 12px 上移并淡入。
- 资料卡：按索引 20～30ms 轻微错峰出现；选中时 160ms 上浮 2px、描边点亮。
- 右侧资料包：新增条目 180ms 从右侧 8px 移入。
- 确认：按钮显示一次短暂确认态，不加入阻塞业务的长动画。
- 遵循 `prefers-reduced-motion`，减少位移动画。

## 9. 验收标准

### 9.1 功能验收

1. 工程师登录后，首页“检修资料”可点击。
2. 编组台能看到案例、知识、手册三类内容和各自数量。
3. 搜索“风扇”能命中相关案例和知识；切换手册能看到厂商手册。
4. 可选中至少 1 个案例、1 条知识和 1 份手册，右侧资料包即时更新。
5. 未点击确认直接关闭时，首页原选择不改变。
6. 点击确认后，首页卡片显示已选数量和资料包标签。
7. 点击“开始接诊”后，所选资料不会丢失。
8. 第 5 步能看到真实选中的资料，而不是固定假文本。
9. 最终摘要同时区分“现场材料”和“诊断参考”。
10. 退出登录后，本次资料包被清空；刷新页面也不承诺恢复。

### 9.2 视觉与可用性验收

1. 1920×1080 演示分辨率下，编组台无需页面级滚动即可完成主要选择操作。
2. 1366×768 下资料墙内部可滚动，确认按钮始终可见。
3. 小于 900px 时左右两栏改为上下布局，不产生横向溢出。
4. 资料卡有默认、悬停、选中、键盘焦点四种状态。
5. Escape、遮罩和关闭按钮均能关闭编组台。

### 9.3 工程验收

- `npm --prefix frontend run build` 通过。
- `git diff --check` 通过。
- 浏览器控制台无本功能新增错误。
- 主流程浏览器验收：首页选择 → 进入接诊 → 第 5 步 → 最终摘要。

## 10. 后续阶段

4. 根据设备、告警和现场描述生成动态推荐；
