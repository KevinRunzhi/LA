# 站控慧眼：简版 PRD 与演示流程说明

## 1. 项目定位

本项目面向第十五届中国软件杯 A1 赛题“基于多模态大模型技术的设备检修知识检索与作业系统”。

系统定位为：

> 面向油气管道场站工控设备检修场景的智能检修辅助系统。

首期不做大而全的工控系统平台，而是收敛到一个核心设备做深：

> 研华 ACP-4000 / IPC-610 工控机整机。

PLC 控制柜、SCADA 监控平台、通信链路、现场监测点和调控中心作为关联对象、影响范围节点和后续扩展保留，不作为首期主诊断设备。PLC 系统专家 Agent 可以保留，当问题涉及 PLC 或影响范围时参与会诊。

系统边界明确为“只读不控”：只做检修知识检索、故障辅助判断、标准化作业指引和经验沉淀，不向 PLC、SCADA、现场仪表或执行机构下发控制指令，不做工控网络安全攻防。

## 2. 首期范围

### 2.1 核心设备

| 类型 | 范围 |
|---|---|
| 主演示设备 | 研华 ACP-4000 / IPC-610 工控机整机 |
| 关联对象 | PLC 控制柜、SCADA 监控平台、通信链路、监测点、调控中心 |
| 后续扩展 | PLC 控制柜完整诊断闭环、更多工控柜/模块 |

### 2.2 故障方向

| 设备 | 故障方向 |
|---|---|
| 工控机整机 | 高温/散热异常 |
| 工控机整机 | 风扇故障/滤网堵塞/风道不畅 |
| 工控机整机 | 供电异常/冗余电源单路异常 |
| 工控机整机 | 硬盘故障/存储异常 |
| 工控机整机 | 指示灯/蜂鸣器/监控告警异常 |
| 工控机整机 | 系统运行异常/死机/数据上传中断 |

主演示优先选择：

> 工控机高温/散热异常。

备用演示选择：

> 工控机供电异常或硬盘/存储异常。

## 3. 要解决的问题

油气管道场站通常少人值守、连续运行，站控系统承担现场数据采集、状态监测、报警展示和远程上传。工控机异常后，可能造成监控告警失效、数据上传中断、历史数据记录异常或现场处置依赖人工确认。

传统检修中存在的问题：

1. 工控机手册、巡检规程、历史案例和老师傅经验分散。
2. 新人遇到温度告警、风扇异常、硬盘指示灯异常时，不知道先查什么。
3. 现场照片、面板指示灯、告警截图、机柜环境无法直接和知识库关联。
4. 大模型直接回答不够可信，需要手册来源、案例证据和专家审核。
5. 检修作业涉及断电、静电防护、禁止热插拔、恢复运行确认等安全项，容易遗漏。
6. 老师傅经验难沉淀，例如“沙尘环境下滤网维护周期要缩短”。

## 4. 核心功能

### 4.1 智能诊断台

用户先描述现场现象，例如：

```text
站控柜内工控机温度告警，风扇声音异常。
```

系统继续追问：

1. 场站位置和机柜位置。
2. 设备型号。
3. 当前检修等级。
4. 是否有工控机面板照片、机柜环境照片或告警截图。

补齐信息后，系统形成结构化诊断上下文。

### 4.2 多模态辅助识别

支持上传：

1. 工控机面板照片。
2. 机柜环境照片。
3. 风扇/滤网/进风口照片。
4. 告警截图。
5. 指示灯或蜂鸣器状态描述。

系统识别设备、可见异常、指示灯状态、环境风险和需要人工确认的内容。识别不确定时，允许人工确认或修改。

### 4.3 RAG 知识检索与证据卡片

系统根据设备型号、故障现象、图片识别结果和检修等级检索知识库。

每条建议必须展示来源：

1. 来源文档名。
2. 页码或章节。
3. 引用片段。
4. 相关故障。
5. 相似案例。

### 4.4 多 Agent 协同会诊

首期 Agent 角色：

| Agent | 作用 |
|---|---|
| 接诊 Agent | 整理用户描述，补齐场站、设备、型号、等级等信息 |
| 知识检索 Agent | 检索手册、规程、案例和图谱证据 |
| 工控设备专家 Agent | 分析工控机故障原因和检查顺序 |
| PLC 系统专家 Agent | 当问题涉及 PLC 或影响范围时参与分析 |
| 安全合规 Agent | 检查断电、静电防护、禁止热插拔、恢复确认等安全要求 |

### 4.5 知识图谱与影响范围

知识图谱展示：

```text
工控机
  -> 散热系统
  -> 高温/散热异常
  -> 滤网堵塞 / 风扇故障 / 风道不畅
  -> 清洁滤网 / 检查风扇 / 检查机柜通风
  -> 影响范围：监控告警可靠性、数据上传稳定性、历史记录完整性
```

图谱不只是解释原因，还要回答“这个故障对场站运行意味着什么”。

### 4.6 标准化作业卡

作业卡包含：

1. 场站位置和机柜位置。
2. 设备型号。
3. 故障现象。
4. 风险等级。
5. 影响范围。
6. 作业前安全检查。
7. 步骤化操作。
8. 恢复运行确认条件。
9. 验收标准。
10. 异常升级条件。
11. 引用来源。

首期只支持 PDF 导出：打开打印版页面，通过浏览器另存为 PDF。

### 4.7 专家审核与知识回流

专家可修正系统回答、作业卡步骤、故障原因、处理措施和图谱关系。

审核通过后，修正内容回流到：

1. 知识库。
2. 知识图谱。
3. 专家 Agent 经验记录。

后续再次遇到类似问题时，系统优先引用已审核经验，并显示“专家修正 · 已审核”。

## 5. 操作演示流程

主演示场景：

> 某输气场站巡检人员发现站控柜内工控机温度告警，风扇声音异常，需要快速判断原因并生成检修作业卡。

### 5.1 登录系统

一线人员登录，进入工作台。

工作台展示：

1. 开始智能诊断入口。
2. 最近诊断记录。
3. 专家待审核数量。
4. 当前主演示设备：研华 ACP-4000 / IPC-610 工控机整机。

### 5.2 发起智能诊断

用户输入：

```text
站控柜内工控机温度告警，风扇声音异常。
```

系统追问：

1. 场站位置。
2. 机柜位置。
3. 设备型号。
4. 检修等级。
5. 是否上传面板照片或机柜环境照片。

用户确认：

```text
设备：研华 ACP-4000 / IPC-610 工控机整机
检修等级：一级检修
```

### 5.3 上传现场图片

用户上传工控机面板照片和机柜环境照片。

系统输出：

1. 设备疑似为站控工控机。
2. 可能存在散热风险。
3. 建议人工确认风扇状态、滤网积尘和机柜通风。

### 5.4 查看检修建议和证据

系统输出：

1. 可能故障：高温/散热异常。
2. 可能原因：滤网堵塞、风扇故障、风道不畅、机柜环境温度过高。
3. 检查方法：查看风扇、检查滤网、检查进出风通道、核对温度告警记录。
4. 处理建议：停机或按规程切换后清洁滤网、检查风扇并复测。
5. 安全提醒：断电、静电防护、禁止带电拆装、恢复后确认数据上传稳定。

每条建议旁边展示证据卡片。

### 5.5 查看多 Agent 会诊

页面展示：

1. 接诊 Agent：确认现场信息和缺失项。
2. 知识检索 Agent：给出命中的手册、案例和知识来源。
3. 工控设备专家 Agent：分析散热异常排查顺序。
4. 安全合规 Agent：检查作业卡安全项。

PLC 系统专家 Agent 保留为可选：当诊断涉及数据上传中断、PLC 通信或站控影响范围时参与分析。

### 5.6 查看知识图谱与影响范围

图谱高亮：

```text
工控机
  -> 散热系统
  -> 高温/散热异常
  -> 滤网堵塞 / 风扇故障
  -> 清洁滤网 / 检查风扇
  -> 影响范围：监控告警可靠性、数据上传稳定性
```

### 5.7 生成作业卡

系统生成作业卡，包括：

1. 故障现象。
2. 风险等级。
3. 影响范围。
4. 安全前置检查。
5. 步骤化操作。
6. 恢复运行确认条件。
7. 验收标准。
8. 异常升级条件。
9. 引用来源。

用户点击“下载 PDF”，通过浏览器打印保存为 PDF。

### 5.8 专家审核与修正

专家发现系统回答中缺少现场经验，例如：

```text
沙尘环境下，站控柜工控机滤网维护周期应从季度检查缩短为月度检查。
```

专家补充修正并审核通过。

### 5.9 再次诊断验证回流效果

再次提出类似问题后，系统命中刚才专家修正内容，并显示：

```text
专家修正 · 已审核
```

## 6. 首期不做内容

1. 移动 App。
2. 复杂工单系统。
3. 实时传感器接入。
4. 本地大模型部署。
5. 本地 OCR。
6. 扫描版 PDF 识别。
7. Word 导出。
8. 后端复杂 PDF 生成。
9. PLC 控制柜完整诊断闭环。
10. 工控网络安全攻防分析。
11. 向 PLC、SCADA、现场仪表或执行机构下发控制指令。
12. 接入真实生产控制网络。

## 7. 技术实现简述

首期采用轻量、可部署的技术路线：

```text
前端：Vue / React + Vite
后端：Node.js API
部署：Nginx + Node.js API
本地数据：SQLite
大模型能力：云端大模型 API
图谱展示：ECharts
```

LoongArch + 银河麒麟目标机只部署前端静态文件、Node.js API、SQLite 或轻量文件库数据和 Nginx，不部署本地大模型、向量数据库、图数据库或复杂 Docker 编排。

---

## 8. 项目亮点

1. 单设备做深：围绕研华 ACP-4000 / IPC-610 工控机整机建立完整诊断闭环。
2. 智能接诊：用户不需要先知道准确故障名称。
3. 多模态输入：支持现场描述、面板照片、机柜环境照片和告警截图。
4. 证据链式 RAG：每条建议都有来源依据。
5. 知识图谱与影响范围：不仅解释故障原因，也说明对场站运行的影响。
6. 标准化作业卡：输出可执行、可确认、可导出的检修步骤。
7. 专家修正回流：老师傅经验可沉淀，后续同类问题优先引用。
8. 只读不控：系统辅助检修决策，不接入真实控制链路。
9. 龙芯平台适配：本地部署轻量化，智能能力云端化，降低国产平台部署风险。

---

## 9. 一句话总结

> 站控慧眼是面向油气管道场站工控设备检修的智能助手：巡检人员描述现场问题并上传工控机照片，系统通过知识库、多 Agent、知识图谱和专家回流机制，生成有依据、可执行、可持续更新的检修作业方案。
