# 老师评审版 PRD 摘要与演示流程

更新时间：2026-07-10

## 1. 产品一句话

面向油气场站一线检修人员的工业设备智能接诊系统：系统将简要现场描述逐步整理为可确认、可编辑、可执行的检修方案，并把完成后的案例审核归档到知识图谱。

## 2. 当前核心问题

旧版本已经有诊断结论和检修向导，但“方案是怎么得到的”仍像固定流程。最新评审重点不是继续增加向导页面，而是让接诊、补充信息、方案生成和人工确认过程可见。

## 3. 核心用户流程

```text
登录
 -> 工业工作台输入现场现象并选择采集材料
 -> 单页动态接诊工作区
 -> Agent 解析现象、匹配位置与设备台账
 -> 动态生成设备和告警补充卡
 -> 工程师逐项确认
 -> 接入摘要
 -> 分析诊断
 -> 生成可编辑预方案
 -> 确认后进入检修向导
 -> 恢复验证与检修记录
 -> 上传总结、审核归档、图谱出现新增知识
```

## 4. 当前 R0/P0 页面要求

- 首页提交后进入一个接诊工作区，不再跳四个整页。
- 中央位置图始终保留并突出站控柜 A01。
- 右侧 Agent 连续输出解析、匹配、检索依据和下一任务。
- 设备信息卡和告警状态卡只在 Agent 需要时出现。
- 字段可真实选择、修改和确认。
- 已完成卡片收缩为摘要，仍可重新编辑。
- 最后才出现接入摘要和“触发诊断”。

## 5. 业务真实性要求

- 主案例可以继续使用研华 ACP-4000 / IPC-610。
- 品牌字段后续必须覆盖油气行业常见品牌。
- 每个结论、判据和步骤尽量绑定手册、章节、页码或知识条目。
- 界面使用“解析、匹配、核对、生成任务”，不使用无法证明的“真实 AI 识别”表述。

## 6. 功能边界

当前真实交互：字段选择、检查项勾选、步骤推进、预方案编辑（后续 P0）、审核状态切换。

当前演示模拟：Agent 输出、设备识别、知识检索、专家接入、图谱回流数据。

当前不做：真实大模型、多模态识别、图数据库、真实通话。

## 7. 给老师看的阶段性演示

R0 完成后演示：

1. 从首页输入工控机高温与风扇异常。
2. 进入单页接诊工作区，位置图不消失。
3. 观看 Agent 自动分析并生成设备任务。
4. 自动填充设备建议后人工确认。
5. 观看 Agent 检索依据并生成告警任务。
6. 修改一个状态再确认，证明不是纯动画。
7. 查看最终接入摘要。
8. 点击触发诊断，稳定进入现有会诊流程。

本轮不需要演示尚未完成的可编辑预方案、多案例和知识回流；这些在后续 P0 增量展示。

## 8. 整体 MVP 验收

- 三个案例可切换并驱动不同步数。
- 预方案编辑结果进入向导。
- 检修完成后能演示上传、审核、归档和图谱新增节点。
- 龙芯主流程与 pad 演示按既定分工运行。
- 断网时主演示路径可用。
