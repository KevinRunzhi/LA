# LA MVP 当前状态总结

更新时间：2026-07-10

本文面向快速了解项目现状；逐项开发事实与验收记录以 `current-development-status.md` 为准。

## 当前阶段

当前正在实施 **R0 / P0：单页动态接诊工作区**。

目标是把首页到分析诊断之间的旧四页接入流程，重构为中央位置图持续存在、Agent 自动分析、设备与告警卡动态生成、用户逐项确认、接入摘要最后生成的单页工作区。

详细规格：`r0-dynamic-intake-workbench-spec.md`。

## 当前定位

项目是面向油气场站工控设备检修的高保真演示型 MVP：

- 主案例为研华 ACP-4000 / IPC-610 高温、风道、滤网和风扇异常。
- 主流程依赖本地预置数据，默认离线可演示。
- Agent、设备识别、专家审核等当前均为演示模拟，不宣称真实智能识别。
- 暂不接真实大模型、真实多模态识别、图数据库或真实音视频通话。

## 已完成并保留

- React 18 + Vite 6 前端骨架。
- Flask API 与前端静态托管能力。
- 登录页、工业工作台、导航和基础页面结构。
- 单案例诊断结论、五步检修向导和恢复验证。
- 检查项勾选、图片联动、知识证据与基础图谱页。
- 检修记录和专家审核的演示骨架。
- 后端失联时的前端兜底数据。
- 接诊 Agent 字符级流式输出、依据列表和完成结果。

## 已做但正在替换
