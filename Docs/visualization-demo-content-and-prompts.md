# MVP 模拟演示可视化内容清单与提示词

更新时间：2026-07-07

本文档用于把当前 MVP 演示流程中“适合做可视化展示”的内容整理出来，方便发给 Claude 分析展示效果，也方便后续交给 Codex 做前端实现。

核心原则：

- 不实现真实三维、真实上传、真实视频、真实模型定位。
- 优先使用“一张预制图片 + 前端覆盖标记/卡片/动画”的方式。
- 展示效果要像系统真的完成了识别、定位、分析、检修引导。
- 技术难度要低，适合 MVP 快速迭代。
- 可视化只服务演示流程，不追求真实工业系统完整能力。

## 1. 当前演示主线

当前 MVP 演示固定为一个场景：

| 项目 | 内容 |
| --- | --- |
| 场景 | 油气管道输气场站 / 站控区域 |
| 设备 | 研华 ACP-4000 / IPC-610 工控机 |
| 位置 | 某输气场站 · 站控柜 A01 |
| 故障 | 工控机高温告警、风道堵塞、散热异常 |
| 现象 | TEMP/FAN 告警、蜂鸣、风扇声音异常、风扇转速偏低 |
| 流程 | 登录 -> 首页输入 -> 异常接入 -> 分析诊断 -> 检修向导 -> 检修记录 -> 专家回流 |

## 3. 重点可视化内容详细说明

### 3.1 场站/工厂态势图 + 故障位置标记

用途：

- 在“描述现场现象”这一步展示系统根据描述初步定位到站控柜 A01 工控机区域。
- 不需要用户确认，不新增流程步骤。
- 只作为辅助展示，让界面更像真实诊断系统。

推荐展示：

- 右侧显示一张“油气输气场站 / 工业厂区 / 站控柜区域”的预制图。
- 图片上覆盖一个脉冲标记点。
- 标记点旁边显示标签：
  - `站控柜 A01`
  - `工控机区域`
- 底部显示一句：
  - `后续诊断将优先围绕该区域组织检修步骤。`

图片来源建议：

| 来源 | 建议 |
| --- | --- |
| AI 生成图片 | 最推荐。生成一张浅色、干净、带站控柜/工控机/厂区线稿感的背景图 |
| 手动绘制/截图 | 可用 PPT、Figma、Excalidraw 画一个简化厂区图后截图 |
| 真实场站照片 | 不建议第一版用真实照片，容易涉及版权和保密，也可能画面杂乱 |
| 公开工业图片 | 可以参考，但建议最终用 AI 生成或自制图，避免版权问题 |

图片风格建议：

- 浅色背景。
- 蓝灰色线稿或轻量 3D 感。
- 不要暗黑科幻风。
- 不要过度 AI 感的紫色渐变。
- 画面要留出一个明显区域用于标记 `站控柜 A01`。
- 图片最好比例为 `16:9` 或 `4:3`，适合放在右侧卡片里。

### 3.2 站控柜 / 工控机局部位置图

用途：

- 在异常接入或检修向导第 1 步中展示当前设备位置。
- 让用户知道诊断对象是站控柜内的工控机，而不是整个 PLC 控制柜或其他设备。

推荐展示：

- 一张“站控柜局部图”。
- 图中标出：
  - `工控机`
  - `前面板`
  - `TEMP/FAN 指示区域`
- 可以用前端覆盖 2-3 个标签，不需要做复杂交互。

图片来源建议：

| 来源 | 建议 |
| --- | --- |
| 产品资料截图 | 可参考 ACP-4000 / IPC-610 资料，但注意不要直接大篇幅使用受版权保护图片 |
| AI 生成 | 推荐生成“工业机柜内工控机前面板”的示意图 |
| 手动标注图片 | 用户后续可以上传一张已标注图片，前端直接显示 |

### 3.3 多 Agent 会诊动态卡片

用途：

- 展示系统正在分析，而不是瞬间给出结论。
- 当前 MVP 已有 Agent 卡片、进度条和逐条输出，后续可以继续优化展示效果。

推荐展示：

- 三个 Agent 卡片依次运行：
  - 分析诊断 Agent
  - 操作合规 Agent
  - 知识检索 Agent
- 每个 Agent 运行 3-5 秒。
- 当前运行中的 Agent 显示流式输出文本。
- 完成后显示“已完成”。
- 所有 Agent 完成后再输出整版诊断结论。

图片/截图需求：

- 不需要图片。
- 用纯前端 UI、进度条和卡片动画即可。

可优化点：

- Agent 卡片上方增加“正在检索知识库 / 正在校验安全要求”等状态。
- 每条流式输出可以有淡入动画。
- 结论生成前显示一个“正在汇总诊断结论”的状态。

### 3.4 诊断证据卡片

用途：

- 让诊断结论看起来有依据，而不是只靠模型一句话。
- 当前已有 KB-001 到 KB-010 的知识条目。

推荐展示：

- 在诊断结论上方或旁边展示 3-4 张证据卡片。
- 卡片内容：
  - `KB-001 ACP-4000 / IPC-610 散热系统结构`
  - `KB-003 风扇 <500 rpm 告警阈值`
  - `KB-004 系统温度 >55°C 与 CPU >70°C 告警阈值`
  - `KB-006 断电、防静电和禁止带电拆装要求`

图片/截图需求：

- 不需要真实图片。
- 用卡片、标签和命中状态展示即可。

### 3.5 检修步骤图片占位

用途：

- 每个检修步骤有对应图片/示意图，增强工程感。
- 当前不做真实图片上传，只做展示位。

推荐展示：

| 检修步骤 | 推荐图片 |
| --- | --- |
| 确认告警与定位设备 | 站控柜/工控机前面板图，标注 TEMP/FAN 区域 |
| 执行安全准备 | 断电、防静电、挂牌作业的安全示意图 |
| 检查机柜环境与风道 | 风道、进风口、出风口的标注图 |
| 检查滤网、风扇和接线 | 滤网、风扇、FAN1/FAN2 接线标注图 |
| 恢复运行与验证 | 状态恢复图，温度/风扇/数据上传恢复正常 |

图片来源建议：

- 优先 AI 生成或自制示意图。
- 不需要逐步真实拍摄。
- 每张图只要表达“当前检查对象在哪里”即可。
- 后续用户手动标注好图片后，直接替换占位图。

### 3.6 风道/滤网/风扇检查示意

用途：

- 这是工控机高温故障最关键的检修环节。
- 演示时比普通文字更容易让人理解。

推荐展示：

- 一张工控机机箱/站控柜内部示意图。
- 覆盖三个标注：
  - `进风口`
  - `滤网`
  - `风扇`
- 风道方向可以用箭头表示。

实现方式：

- 最简单：一张图 + 前端绝对定位标签。
- 更像系统：标签轻微浮动，当前检查项对应标签高亮。
- 不做真实热点点击。

### 3.7 阈值和恢复验证状态

用途：

- 在恢复验证步骤展示“故障是否恢复正常”。
- 当前是预设演示，可以展示成假数据恢复。

推荐展示：

| 指标 | 异常状态 | 恢复状态 |
| --- | --- | --- |
| 风扇转速 | < 500 rpm | > 500 rpm |
| 系统温度 | > 55°C | ≤ 55°C |
| CPU 温度 | > 70°C | ≤ 70°C |
| 数据上传 | 不稳定/待确认 | 稳定 |
| 观察时间 | 未完成 | ≥ 10 分钟 |

展示方式：

- 小型状态卡片。
- 进度条或绿色恢复标识。
- 不需要接真实传感器。

### 3.8 知识图谱节点图

用途：

- 赛题中提到知识图谱，需要保留并展示出来。
- 当前已有节点图。

推荐优化：

- 选中“工控机”时突出显示关联节点。
- 选中“高温告警”时突出显示可能原因。
- 选中“风扇异常”时显示风扇转速阈值证据。
- 不需要真实图数据库，继续用 JSON 和前端布局即可。

### 3.9 作业卡/检修记录预览

用途：

- 形成“检修闭环”的产品感。
- 演示时可以展示系统自动生成记录。

推荐展示：

- 类似 PDF 表单的预览卡片。
- 包含：
  - 记录编号
  - 设备信息
  - 故障现象
  - 诊断结论
  - 检修步骤完成情况
  - 安全确认
  - 专家审核状态
- 按钮：
  - `打印作业卡`
  - `导出 PDF`
  - `提交专家审核`

技术实现：

- 前端卡片即可。
- 打印可用 `window.print()`。
- PDF 导出可以后续再做。

### 3.10 专家修正回流提示

用途：

- 演示“专家经验沉淀回知识库”。
- 不需要真实专家系统。

推荐展示：

- 专家审核后出现一张回流卡片：
  - `专家修正 · 已审核`
  - `沙尘环境下滤网维护周期从季度缩短为月度`
  - `已写入知识条目 / 图谱关系 / 后续诊断建议`

技术实现：

- 纯前端状态 + 后端模拟接口。
- 不需要数据库。

## 5. 图片来源与截图建议

### 5.1 AI 生成图片建议

适合生成：

- 输气场站/站控室/工业机柜的浅色示意图。
- 工控机前面板示意图。
- 机柜风道、滤网、风扇检查示意图。

生成风格建议：

- 工业运维系统风格。
- 浅色背景。
- 蓝灰线稿。
- 轻微 3D 感，但不要真实复杂 3D。
- 画面干净，方便覆盖标签和标记点。
- 不要出现复杂人物。
- 不要出现真实品牌 logo。

### 5.2 截图/自制图片建议

适合来源：

- PPT/Figma/Excalidraw 自画示意图后截图。
- 使用 AI 生成底图后手动标注。
- 使用设备资料作为参考，但最终图片最好重绘或 AI 生成。

注意：

- 不建议直接使用真实工厂照片作为 MVP 主图。
- 不建议直接复制厂商资料里的完整产品图作为公开演示主视觉。
- 如果使用真实资料截图，最好只作为内部参考，最终展示图用自制或 AI 生成版本。

### 5.3 展示方式建议

推荐使用：

- 图片作为卡片背景。
- 前端用绝对定位叠加标记点、标签、箭头。
- 当前步骤对应标签高亮。
- 使用轻微脉冲动画，不做复杂交互。

不推荐：

- 真三维建模。
- Three.js。
- 可旋转 3D 模型。
- 复杂热点交互。
- 真实视频流。
- 真实图像识别。

## 9. 不建议当前阶段实现的内容

当前阶段不建议做：

- 真 3D 场景。
- Three.js 旋转模型。
- 真实设备热点选择。
- 真实图片上传和图像识别。
- 真实视频输入。
- 真实语音识别。
- 真实传感器数据。
- 真实图数据库。
- 复杂权限和多角色可视化。

原因：

- 技术成本高。
- 与当前 MVP 演示目标不匹配。
- 容易拖慢开发节奏。
- 低成本图片化展示已经足够支撑比赛第一版效果。

## 10. 当前最推荐的落地顺序

建议按以下顺序推进：

1. 先确定 IMG-001 场站态势图。
2. 在“描述现场现象”页右侧替换当前 CSS 占位图。
3. 增强检修向导图片占位，先统一视觉风格。
4. 准备 IMG-004 风道图和 IMG-005 风扇滤网图。
5. 在检修向导第 3、4 步替换对应图片。
6. 优化诊断结论证据卡片。
7. 最后再优化作业卡预览和专家回流提示。

最低成本版本只做第 1 和第 2 步即可。
