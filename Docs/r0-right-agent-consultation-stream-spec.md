# R0【P0】右侧多 Agent 会诊消息流改造 Spec

> 版本：V1.0
> 日期：2026-07-11
> 前置回滚点：`908f1af feat: rebuild dynamic intake workspace`
> 实现边界：React 前端高保真模拟，不新增依赖，不新增后端接口

## 1. 改造目标

将“多 Agent 会诊”从中央业务工作区完整迁移到右侧辅助对话区，使中央区域负责业务状态和最终结论，右侧负责 Agent 思考、依据检索、阶段输出和会诊衔接。

## 2. 完成定义

1. `analysis` 阶段中央不再渲染原 Agent 进度轨、Agent 卡片列表和流式分析行；
2. 中央只保留轻量的“会诊进行中”业务状态、已确认事件对象和结果去向，不复制右侧过程；
3. 右侧标题在 `analysis` 阶段切换为“多 Agent 会诊”；
4. 三个 Agent 按既有顺序依次运行，已完成、运行中、等待中状态清晰；
5. 每个 Agent 至少展示职责、输入、分析动作、检索依据和阶段结论五类信息；
6. 当前 Agent 的内容逐条淡入，并包含思考脉冲、检索动画、流式光标和 Agent 交接动画；
7. 已完成 Agent 保留紧凑记录，当前 Agent 保持展开，未开始 Agent 只显示等待条；
8. 所有 Agent 完成后显示证据汇合和“正在生成诊断结论”；
9. 右侧消息容器在新增内容、文字增长、Agent 切换和完成时自动平滑滚动到底部；
10. 输入框固定在右侧底部，内容滚动不影响输入框位置；
11. 用户手动发送追问后的流式回复同样自动跟随到底部；
12. `prefers-reduced-motion` 下停用位移动画，但内容和状态仍完整可读。

## 3. 信息架构

```text
右侧辅助区域（analysis 阶段）
├── 会诊标题与总体进度
├── 会诊消息流（独立滚动）
│   ├── 事件输入摘要
│   ├── 分析诊断 Agent
│   │   ├── 思考过程
│   │   ├── 输入事实
│   │   ├── 分析动作
│   │   ├── 引用依据
│   │   └── 阶段结论
│   ├── 知识检索 Agent
│   ├── 操作合规 Agent
│   └── 证据汇合 / 结论生成状态
└── 固定追问输入框
```

## 4. Agent 展示内容

### 4.1 分析诊断 Agent

- 输入：异常描述、设备型号、TEMP/FAN、温度和转速；
- 动作：拆分现象、关联故障部件、形成候选原因；
- 输出：优先进入散热异常诊断方向。

### 4.2 知识检索 Agent

- 输入：候选原因和设备对象；
- 动作：检索设备手册、阈值规则、历史检修案例；
- 输出：命中风道、滤网、风扇和温度阈值依据。

### 4.3 操作合规 Agent

- 输入：诊断方向与计划检查动作；
- 动作：校验断电、挂牌、防静电和恢复验证要求；
- 输出：形成可进入检修预方案的安全边界。

## 5. 状态与动画

```text
WAITING → THINKING → RETRIEVING → SYNTHESIZING → DONE
```

- Agent 激活：头像/编号缩放进入，状态点脉冲；
- 思考内容：每 `360–520ms` 生成一条；
- 依据检索：扫描线和加载图标持续，命中后转为勾选；
- 阶段结论：最后出现，随后当前卡收缩为已完成记录；
- Agent 交接：显示“已将阶段结果交给下一 Agent”；
- 全部完成：三路证据连线汇合，出现结论生成状态。

## 6. 自动滚动规则

右侧滚动容器使用 `ref` 管理：

```js
scrollTo({ top: scrollHeight, behavior: "smooth" })
```

触发条件：

- 会诊阶段进入；
- 当前 Agent 改变；
- 当前 Agent 可见内容数量增加；
- Agent 阶段结论出现；
- 普通辅助消息新增；
- 流式回复文字长度增加；
- 检索依据数量增加。

滚动目标是 `.assistant-body` 底部；输入框不放入滚动容器。

## 7. 中央工作区

中央 `analysis` 页面只展示：

- 当前业务状态：“多 Agent 会诊进行中”；
- 已确认事件对象、地点和诊断方向；
- 提示：“详细会诊过程正在右侧生成”；
- 简洁总体进度和三个 Agent 状态点。

不得再展示完整 Agent 文本、检索依据或重复会诊卡片。

## 8. 验收路径

1. 从“现场异常事件全景”启动诊断；
2. 中央原会诊动画列表消失；
3. 右侧依次运行三个 Agent，内容明显多于原辅助对话提示；
4. 每个 Agent 的思考、检索、依据和结论按动画出现；
5. 观察整个过程中右侧自动跟随最新内容；
6. 三个 Agent 完成后自动进入诊断结论；
7. 检查浏览器控制台无错误；
8. Node `20.19.4` 构建通过。
