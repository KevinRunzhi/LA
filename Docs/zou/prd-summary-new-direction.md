## 一、产品定位

**一句话定位**：一套运行在龙芯平台上的工控设备异常检测与故障分析系统——现场工程师用平板跟随三维可视化检修向导逐步排障，后台沉淀专家修正后的检修案例，让每一次维修都变成下一次的知识。

**设计三原则（新）**：
1. **步骤为主体**：产品主界面是检修向导（第几步、干什么、打勾确认），不是聊天窗；对话助手固定在右下角，仅作辅助。
2. **可视化优先，多形态反馈**：能用厂区定位图、部件图示、动画、语音播报表达的，不用大段文字；"交互式 = 有点击有反馈，且反馈是多种形式的"（老师原话）。
3. **AI 离线生产、系统预置呈现**：步骤方案、图示素材、播报语音由大模型在开发期生产、人工审定后入库；运行时以匹配调用为主、实时生成为辅——保证工业内容可靠与演示零网络依赖。

**用户角色**：
- 一线检修工程师（平板向导端）：报障、看定位、跟向导干活、完成检修。
- 后台管理员 / 专家（龙芯菜单后台端）：知识库管理、案例审核修正、用户角色部门与基础数据管理。

**系统边界**：对设备只读不控；不做工控网络安全攻防；数据全脱敏。

## 二、功能分解（三级树状，老师钦定七模块框架，功能名一律用短语）

```text
工控设备异常检测与故障分析系统
├─ 1 异常检测模块
│   ├─ 1.1 多模态报障（文字录入 / 语音报障 / 现场拍照 / 视频上传）
│   ├─ 1.2 设备数据接入（工控机状态读取·模拟数据面板）
│   └─ 1.3 异常归类（异常类型识别、影响初判）
├─ 2 故障分析模块
│   ├─ 2.1 知识检索分析（手册检索、案例检索、证据溯源）
│   ├─ 2.2 专家 Agent 分析（诊断推理、原因排序、方案推荐）
│   └─ 2.3 远程专家连线（语音会议、视频会议·模拟，预留 IM 接口）
├─ 3 工控机检修知识库
│   ├─ 3.1 手册资料库（文档入库、切块检索）
│   ├─ 3.2 知识图谱（设备-部件-故障-案例关系、影响范围查询）
│   └─ 3.3 维修案例库（历史案例、专家修正案例、金标标记）
├─ 4 大模型管理模块
│   ├─ 4.1 接诊检索 Agent（要素抽取、知识召回）
│   ├─ 4.2 诊断分析 Agent（故障判断、步骤方案匹配）
│   ├─ 4.3 操作合规 Agent（安全项校验·仅引用安全规则库）
│   └─ 4.4 API 与调用管理（模型配置、调用日志、缓存管理）
├─ 5 可视化模块
│   ├─ 5.1 厂区定位图（三维布局展示、故障设备高亮）
│   ├─ 5.2 部件可视化（部位图示、步骤高亮联动）
│   └─ 5.3 过程动效（检索动画、进度动效）
├─ 6 检修作业向导模块
│   ├─ 6.1 步骤向导（分步展示、逐项打勾、步骤增删）
│   ├─ 6.2 语音播报（分步语音提示·预制 TTS）
│   └─ 6.3 检修记录（过程留痕、记录导出、提交审核）
└─ 7 系统管理模块（后台）
    ├─ 7.1 用户管理（用户、角色、部门）
    ├─ 7.2 基础数据（设备台账、场区配置、安全规则库）
    └─ 7.3 知识库管理（资料上传、案例审核、图谱维护）
```

## 三、核心业务流程

### 3.1 主流程（现场端）
```text
报障（说一句 / 打一行 / 拍一张 / 读数据）
 → 异常检测归类
 → 故障分析：检索(手册+案例) + 三 Agent 结论 + 厂区图定位故障设备
 → 展示分析结果页（三 Agent 分栏、证据编号、定位高亮、安全项橙块）
 → 进入检修向导（推荐 4~6 步方案；用户可增删步骤后确认）
 → 逐步执行：每步 = 部件图高亮 + 操作说明 + 语音播报 + 打勾
 → 全部完成 → 生成检修记录 → 提交后台审核（可选导出记录）
```

### 3.2 回流流程（后台端）
```text
专家登录龙芯后台 → 案例审核列表 → 打开案例左右对照 → 修改补充经验
 → 通过 → 案例入知识库（带"专家修正"标记）+ 图谱新增边
 → 下次同类故障：检索命中修正案例，前端出金色标记
```

### 3.3 右下角对话助手（辅助流程）
- 任意页面可唤起；用于补充描述、追问"这一步为什么要这样做"（回答仅基于当前案例已有结论与证据，不引入新来源）。
- 保留旧方案槽位思想的简化版：助手抽取的要素直接回填报障表单，不再做多轮长追问。

## 五、非功能需求

- **平台**：前后端均须在 LoongArch（龙芯）+ 银河麒麟环境运行；前端 Node 开发态直跑，后端 Python 直跑，SQLite 免编译。
- **离线兜底**：预置案例全套内容（分析/步骤/图/语音）离线可用，主演示流程零网络依赖。
- **性能**：向导页操作零等待；分析页允许 3~8 秒并配检索动画。
- **可信**：结论带出处编号；安全项仅来自安全规则库；步骤方案人工审定入库。
- **开发过程要求（老师明令）**：迭代式开发，每完成一个迭代必须在龙芯环境跑通一次；首个迭代为"仅主界面空壳"。
