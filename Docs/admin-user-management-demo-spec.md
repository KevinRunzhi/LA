# 用户管理员演示门户 Spec

## 1. 目标

为系统补充一个职责单一的用户管理员角色，用于演示平台如何管理工程师和专家账号。

管理员门户不负责故障诊断、案例审核、检修知识编辑或知识图谱维护，仅表达账号和组织管理能力。

## 2. 演示账号

- 登录账号：`admin`
- 角色名称：用户管理员
- 展示姓名：系统管理员
- 所属团队：平台运维中心
- 当前登录仍沿用 MVP 演示映射，不校验真实密码。

## 3. 信息架构

管理员登录后进入独立门户，不显示工程师诊断台和专家知识后台。

左侧菜单仅包含：

- 用户管理
- 当前账号摘要
- 退出登录

管理员页面不提供案例、知识库、知识图谱或专家审核入口。

## 4. 页面结构

### 4.1 顶部说明

- 页面标题：用户与角色管理
- 辅助说明：维护工程师和专家的演示账号、组织归属与启停状态
- 主操作：新增用户

### 4.2 统计摘要

展示四项实时统计：

- 工程师数量
- 专家数量
- 正常账号数量
- 停用账号数量

### 4.3 筛选与搜索

- 支持全部、工程师、专家三种角色筛选。
- 支持按姓名、账号、场站、班组、机构和专业方向搜索。

### 4.4 用户列表

每个用户展示：

- 姓名与用户编号
- 登录账号
- 角色
- 工程师：所属场站、班组
- 专家：所属机构、专业方向
- 账号状态
- 编辑、启用/停用、重置演示密码操作

## 5. 用户操作

### 5.1 新增与编辑

- 必填字段：姓名、登录账号、角色。
- 工程师填写场站和班组。
- 专家填写机构和专业方向。
- 登录账号在当前演示数据中必须唯一。

### 5.2 启用与停用

- 点击后立即更新列表和统计摘要。
- 停用仅改变演示状态，不影响当前已登录页面。

### 5.3 重置密码

- 不生成或存储真实密码。
- 点击后显示“演示密码已重置为 123456”的结果提示，用于汇报演示。

## 6. 数据策略

- 首次进入优先读取现有 `/api/admin/users` 人员种子。
- 后端不可用时使用前端内置人员兜底。
- 新增、编辑和启停结果写入浏览器 `localStorage`，刷新页面后保留。
- 不新增真实认证、密码哈希、权限令牌或生产数据库表。

## 7. 权限边界

- 管理员只能管理工程师和专家，不显示管理员自身的编辑操作。
- 管理员不能修改专业知识、案例结论或图谱关系。
- 工程师和专家原有入口与流程保持不变。

## 8. 验收流程

1. 使用 `admin` 登录并进入独立用户管理门户。
2. 确认页面没有案例、知识库和知识图谱菜单。
3. 新增一名工程师并在列表中看到该账号。
4. 编辑该工程师的场站或班组。
5. 停用账号，确认统计数字和状态同步变化。
6. 点击重置密码，确认出现演示结果提示。
7. 刷新页面，确认本地演示修改仍然存在。
8. 退出后使用工程师或专家账号，确认原流程不受影响。
