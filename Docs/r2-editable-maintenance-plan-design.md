# R2【P0】千问式可编辑检修预方案设计

> 版本：V1.0
> 日期：2026-07-10
> 状态：用户已确认，进入 Spec 与开发阶段

## 1. 产品目标

R2 位于“诊断结论”和“检修向导”之间。系统先生成一份两级检修大纲初稿，工程师直接在大纲上增删修改，确认后才成为正式检修方案。

核心表达：

> 系统负责生成建议，工程师拥有最终修改权；确认后的方案真实驱动检修向导。

本页不是普通步骤列表，也不是复杂表单，而是类似千问任务大纲的直接编辑体验。

## 2. 目标流程

```text
多 Agent 会诊
  -> 诊断结论
  -> 点击“生成检修预方案”
  -> 检修编排 Agent 逐步生成大步骤与小维修步骤
  -> 进入预方案确认页
  -> 工程师增删改与调整顺序
  -> 确认检修方案
  -> 检修向导按确认后的大纲执行
```

## 3. 两级大纲结构

### 3.1 大步骤

大步骤表示完整检修阶段，例如：

1. 确认异常与定位设备；
2. 执行安全准备；
3. 检查机柜环境与风道；
4. 检查滤网、风扇和接线；
5. 恢复运行与验证。

每个大步骤显示编号、标题、简短目标、来源依据、生成/修改标识以及展开状态。

### 3.2 小维修步骤

小维修步骤是可执行动作，例如：

- 检查 TEMP/FAN 指示灯；
- 记录风扇转速；
- 检查进风口是否遮挡；
- 清理门滤网；
- 核对 FAN1/FAN2 接线；
- 连续观察运行状态 10 分钟。

小维修步骤必须显示在所属大步骤下，保持清晰父子层级。

## 4. 生成体验

点击“生成检修预方案”后，右侧检修编排 Agent 依次输出：

1. 读取诊断结论和原因排序；
2. 匹配设备手册、检修规程与已审核案例；
3. 核对断电、防静电等安全前置条件；
4. 生成大步骤标题；
5. 在每个大步骤下逐条生成小维修步骤；
6. 进入“等待工程师确认”。

大纲不能瞬间全部出现，应按“大步骤 -> 小维修步骤”的顺序逐项浮现。

## 5. 页面布局

继续保留当前三栏工作台：

- 左侧：可折叠诊断流程；
- 中间：连续的两级检修大纲编辑区；
- 右侧：检修编排过程、依据和工程师修改影响。

中间区域不使用大量互不关联的卡片，而是一份带垂直层级线的连续大纲。

## 6. 编辑行为

### 6.1 文字修改

- 点击大步骤标题、小维修步骤文字后原位进入编辑；
- `Enter` 保存，`Esc` 取消，失焦保存；
- 修改后标记“工程师修改”。

### 6.2 增删

- 每个大步骤下提供“添加小维修步骤”；
- 大纲底部提供“新增检修阶段”；
- 普通步骤支持删除；
- 删除大步骤时提示其子步骤将一并删除；
- 删除后提供短时撤销提示。

### 6.3 顺序

第一版使用“上移/下移”，同时支持大步骤和同组小维修步骤。暂不做拖拽。

## 7. 安全边界

正常关机、拔除电源、等待冷却、佩戴防静电手环、禁止带电拆装等强制安全项显示锁定标记。

- 可查看和补充说明；
- 不允许直接删除；
- 不允许移动到拆检动作之后；
- 删除包含强制安全项的大步骤时必须阻止并解释原因。

## 8. 来源与修改标识

系统内容显示来源类型：设备手册、检修规程、已审核案例、操作合规 Agent、诊断结论。

工程师操作显示：工程师修改、工程师新增、工程师删除。右侧辅助区同步记录操作及其对向导的影响。

## 9. 三份状态

```js
systemGeneratedPlan // 系统初稿，始终保留
workingPlan         // 当前编辑版本
confirmedPlan       // 最终确认并传给向导的版本
```

“恢复系统初稿”用系统版本覆盖工作版本；“确认方案”复制工作版本为确认版本。检修向导只能读取 `confirmedPlan`。

## 10. 底部操作区

- 返回诊断结论；
- 恢复系统初稿；
- 当前阶段数、维修步骤数和修改数；
- 确认方案并进入检修向导。

确认条件：至少一个大步骤；每个大步骤有标题和至少一个小维修步骤；全部强制安全约束仍存在。

## 11. 第一版范围

### 必须实现

- 两级大纲逐项生成；
- 大步骤和小维修步骤增、删、改；
- 上移、下移；
- 安全项锁定；
- 恢复系统初稿；
- 确认后真实传入检修向导；
- 左侧流程、向导步数和检查项同步；
- 右侧记录生成依据与工程师修改。

### 暂缓

- 拖拽排序；
- 完整撤销历史；
- 多版本对比；
- 后端持久化和多人协作；
- 大模型局部重生成；
- Word/PDF 导出；
- 依据与图片联动高亮。

## 12. 现场演示脚本

1. 展示系统逐步生成两级大纲；
2. 删除一个普通小维修步骤；
3. 新增“拍摄滤网积尘照片”；
4. 修改一个大步骤标题；
5. 新增“准备备用风扇”阶段；
6. 确认方案；
7. 在向导中展示新增内容已生效；
8. 强调系统生成建议、工程师最终确认。
