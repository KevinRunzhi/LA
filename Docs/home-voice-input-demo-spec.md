# 首页语音输入演示功能 Spec

## 1. 功能定位

在工程师首页“描述现场现象”区域提供一个纯视觉的语音输入演示入口，用于汇报时表达系统具备语音接入形态。

本功能不接入平台语音能力，不申请麦克风权限，不采集、存储或上传音频，也不向现场现象输入框写入任何内容。

## 2. 入口与布局

- 入口位于“开始接诊”按钮左侧。
- 入口使用独立圆形麦克风图标按钮，不显示可见文字。
- 麦克风按钮与“开始接诊”组成紧凑操作组，保持原输入框宽度和主按钮层级。
- 默认状态使用低亮度蓝灰色描边；活动状态使用工业蓝高亮和克制的呼吸光晕。

## 3. 交互流程

### 3.1 待机

- 显示静止麦克风图标。
- 悬停提示为“语音输入演示”。
- 无任何后台任务或媒体权限请求。

### 3.2 正在语音输入

- 点击按钮后立即进入活动状态。
- 演示持续 3.6 秒，属于用户要求的 3～4 秒区间。
- 麦克风图标在按钮内部轻微上下浮动。
- 图标两侧的三组短声波错峰伸缩。
- 按钮边缘出现低强度蓝色呼吸光晕。
- 再次点击可提前结束演示并恢复待机。

### 3.3 自动结束

- 3.6 秒后自动恢复待机状态。
- 不显示识别结果，不修改输入框，不自动开始接诊。
- 用户仍需手动点击“开始接诊”进入现有流程。

## 4. 状态与无障碍

- 组件仅包含 `idle` 和 `listening` 两种本地 UI 状态。
- 使用 `aria-pressed` 表达按钮是否处于活动状态。
- 使用动态 `aria-label` 提供“开始/停止语音输入演示”的读屏说明。
- 使用仅供读屏读取的状态文本通知演示开始和结束。
- 在 `prefers-reduced-motion` 环境下关闭位移动画，保留颜色和状态变化。

## 5. 技术边界

- 不调用 `navigator.mediaDevices`、`MediaRecorder`、Web Speech API 或任何音频播放接口。
- 不新增第三方依赖。
- 不改变首页输入值、接诊提交函数、快捷接诊数据或后端接口。
- 动画使用 CSS keyframes；3.6 秒自动结束使用 React 定时器，并在组件卸载时清理。

## 6. 检修智能体入口

- 检修智能体底部输入区复用同一个语音输入演示组件。
- 麦克风按钮位于聊天输入框和发送按钮之间，即发送按钮左侧。
- 聊天入口使用紧凑变体：占用一个 36 px 网格列，圆形按钮尺寸为 30 px。
- 动画时长、提前结束行为、无媒体 API 边界与首页入口完全一致。
- 两个入口各自维护本地状态，互不触发，也不会修改聊天草稿或发送消息。

## 7. 验收标准

1. 麦克风按钮位于“开始接诊”左侧且无可见文字。
2. 点击后图标、声波和光晕持续动画约 3.6 秒。
3. 动画结束后按钮自动恢复待机。
4. 活动期间再次点击可以提前结束。
5. 点击及自动结束均不会改变现场现象输入框。
6. 不出现麦克风权限请求，也不产生音频数据。
7. 键盘焦点清晰，读屏状态可识别，减少动态偏好生效。
8. 检修智能体输入框与发送按钮之间显示同款紧凑麦克风按钮。
9. 检修智能体语音演示结束前后，聊天草稿和发送按钮逻辑保持不变。
