## 2. R0 / P0 定义

R0 对应 2026-07-10 Meeting 中第一项 P0 需求的当前落地批次（Meeting 原编号 R1）。详细完成定义以 `r0-dynamic-intake-workbench-spec.md` 为准：把首页到分析诊断之间的旧四页接入流程改造成单页动态接诊工作区。

验收目标：

- 点击“开始接诊”后不能直接进入固定四步页面。
- 中央故障位置图在整个接入阶段持续显示。
- Agent 自动流式解析现象、定位区域、匹配台账和核对依据。
- 设备信息卡、告警状态卡按状态机依次生成，未生成的卡片完全隐藏。
- 每一项都由用户确认或修改，系统不替代工程师作最终确认。
- 已完成卡片收缩为摘要并允许重新编辑。
- 接入摘要最后生成，点击后稳定进入现有分析诊断流程。
- WSL 构建通过，并在龙芯 Node 开发态完成同模块验证后，R0 才能标记完成。

- 浏览器验证 PLC/通信分支的任务卡、右侧 Agent 轨迹和接入摘要同步更新；摘要页会阻止该分支误入尚未建设的散热诊断结论。
