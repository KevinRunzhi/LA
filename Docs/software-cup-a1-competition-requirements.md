## 赛题简介

本赛题聚焦工业、能源、制造等领域设备检修的核心痛点，以多模态大模型技术为核心支撑，引导参赛者开发一款兼具智能化、实用性与创新性的设备检修知识检索与作业辅助系统。

赛题旨在推动大模型技术与行业场景深度融合，破解传统设备检修中知识分散、检索低效、作业不规范、经验传承难等问题，打造可落地、高价值的行业智能助手，助力行业数字化转型与效率升级。

## 赛题业务场景

在钢铁、汽车制造等行业，大型企业产线设备密集且联动性强，设备稳定运行直接决定产能与效益。企业在传统运维中存在以下痛点：

- 检修知识分散于手册、日志与老员工经验中，形成“数据孤岛”；
- 新人上手慢；
- 非计划停机单次损失超百万元；
- 经验传承难；
- 故障响应慢。

基于上述业务背景，赛题聚焦制造业核心生产设备检修。参赛者可基于本地大模型或 API（如 Qwen、DeepSeek 等），开发适配现场检修作业的系统，整合多模态检修资源，辅助一线人员快速检索方案、诊断故障，实现知识数字化沉淀与高效复用。

## 基本功能需求

系统需采用 B/S 架构开发，软件需部署在自主指令系统 LoongArch 架构与银河麒麟高级服务器版上运行。不满足该要求视为 0 分。

### 1. 大模型服务与可视化界面

支持本地部署大模型服务或云端大模型服务，具备 PC Web 或 App 可视化界面，方便用户访问与操作。

### 2. 多模态知识检索功能

支持文本、故障图片、设备型号等多类型信息输入，实现精准语义检索与跨模态匹配，快速调取检修手册等资源。

### 3. 标准化作业指引功能

嵌入检修全流程，提供步骤化操作指引、合规校验提醒，支持根据设备类型与检修等级个性化推送流程，助力规范作业、降低操作失误率。

### 4. 知识沉淀与更新功能

支持一线人员上传检修案例、经验总结等内容，经审核后纳入知识图谱，实现知识实时更新。同时支持手动标注与修正大模型输出结果，优化系统适配性。

## 非功能性需求

- 软件界面设计合理、美观大方；
- 软件开发文档完整规范；
- 人机交互操作便捷，适合用户使用；
- 软件保持易用性与稳定性。
